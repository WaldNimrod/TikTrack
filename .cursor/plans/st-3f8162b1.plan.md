# תוכנית עדכון גרסה בפרודקשן - עם מיגרציות וניהול שרת

## מטרה

לעדכן את סביבת הפרודקשן עם כל השינויים מ-main, להריץ מיגרציות נדרשות, לעדכן ולהפעיל את השרת על פורט 5001, ולבצע בדיקות מקיפות.

## שלבים מקדימים (בסביבת הפיתוח)

### שלב 0.1: בדיקת מוכנות

- [ ] בדיקת שינויים לא שמורים ב-main
- [ ] Push כל השינויים ל-main
- [ ] יצירת changelog (אופציונלי)

### שלב 0.2: זיהוי מיגרציות נדרשות

**קובץ:** `scripts/production-update/utils/migration_detector.py` (חדש - ליצור)

**תפקיד:**

1. סריקת `Backend/migrations/` למיגרציות חדשות (לפי תאריך שינוי)
2. השוואת מבנה DB בין dev ל-production:

   - בדיקת טבלאות חסרות
   - בדיקת עמודות חסרות
   - השוואת indexes

3. זיהוי מיגרציות שצריך להריץ לפי:

   - טבלאות שהמיגרציה יוצרת (לא קיימות ב-production)
   - עמודות שהמיגרציה מוסיפה (לא קיימות ב-production)

4. יצירת רשימת מיגרציות בסדר נכון (dependencies)

**פלט:**

- רשימת מיגרציות שצריך להריץ
- סדר הרצה מומלץ
- אזהרות על מיגרציות שעלולות לגרום לבעיות

### שלב 0.3: זיהוי קבצים חדשים

**קובץ:** `scripts/production-update/utils/new_files_detector.py` (חדש - ליצור)

**תפקיד:**

1. השוואת `Backend/` ל-`production/Backend/`
2. זיהוי קבצים חדשים שלא קיימים ב-production
3. זיהוי קבצים שהשתנו משמעותית
4. בדיקת קבצים קריטיים (models, routes, services)

**פלט:**

- רשימת קבצים חדשים
- רשימת קבצים שהשתנו
- קבצים קריטיים שצריך לבדוק ידנית

## שלבים בפרודקשן

### שלב 1: שמירת שינויים בפרודקשן

**קובץ:** `scripts/production-update/steps/00_save_production_changes.py`

**פעולות:**

1. בדיקת שינויים לא שמורים ב-production branch
2. שמירת שינויים (commit)
3. Push ל-production branch

### שלב 2: בדיקת מצב השרת

**קובץ:** `scripts/production-update/steps/02_check_server.py` (חדש - ליצור)

**פעולות:**

1. בדיקת אם השרת רץ על פורט 5001:
   ```bash
   lsof -i :5001
   curl -f http://127.0.0.1:5001/api/health
   ```

2. זיהוי PID של השרת (אם רץ)
3. בדיקת health check:

   - `GET http://127.0.0.1:5001/api/health`
   - בדיקת תגובה HTTP 200

4. דיווח על מצב השרת:

   - רץ/לא רץ
   - גרסה נוכחית (אם זמינה)
   - זמן פעולה (uptime)

**פלט:**

- מצב השרת (רץ/לא רץ)
- PID (אם רץ)
- תוצאות health check

### שלב 3: גיבוי PostgreSQL

**קובץ:** `scripts/production-update/steps/04_backup_database.py` (לעדכן)

**פעולות:**

1. יצירת גיבוי PostgreSQL:
   ```bash
   pg_dump -h $POSTGRES_HOST -p $POSTGRES_PORT -U $POSTGRES_USER -d $POSTGRES_DB > _Tmp/backups/postgresql_backup_$(date +%Y%m%d_%H%M%S).sql
   ```

2. אימות גיבוי (checksum)
3. שמירת נתיב הגיבוי

**שינוי נדרש:**

- עדכון `04_backup_database.py` לתמוך ב-PostgreSQL במקום SQLite

### שלב 4: עדכון main branch

**קובץ:** `scripts/production-update/steps/01_update_main.py`

**פעולות:**

1. `git fetch origin main`
2. `git merge origin/main` (או `git pull origin main`)

### שלב 5: מיזוג main → production

**קובץ:** `scripts/production-update/steps/03_merge_to_production.py`

**פעולות:**

1. `git checkout production`
2. `git merge main --no-edit`
3. פתרון קונפליקטים (אם יש)

### שלב 6: סינכרון קוד

**קובץ:** `scripts/production-update/steps/05_sync_code.py`

**פעולות:**

1. הרצת `scripts/sync_to_production.py` (Backend)
2. הרצת `scripts/sync_ui_to_production.py` (UI)
3. דיווח על קבצים שהועתקו

### שלב 7: עצירת השרת (אם רץ)

**קובץ:** `scripts/production-update/steps/07_stop_server.py` (חדש - ליצור)

**פעולות:**

1. בדיקת אם השרת רץ (מתוך שלב 2)
2. אם רץ:

   - שליחת SIGTERM לשרת (graceful shutdown)
   - המתנה עד 30 שניות לסגירה
   - אם לא נסגר - שליחת SIGKILL
   - אימות שהשרת נעצר (בדיקת פורט 5001)

3. דיווח על עצירת השרת

**שימוש:**

- קריאה ל-`server_lock_manager.py` לעצירה בטוחה
- או `kill` ישיר אם יש PID

### שלב 8: זיהוי והרצת מיגרציות

**קובץ:** `scripts/production-update/steps/08_run_migrations.py` (חדש - ליצור)

**פעולות:**

1. הרצת `migration_detector.py` לזיהוי מיגרציות נדרשות
2. הצגת רשימת מיגרציות למשתמש לאישור
3. הרצת כל מיגרציה בסדר הנכון:

   - גיבוי DB לפני כל מיגרציה (אופציונלי)
   - הרצת מיגרציה:
     ```python
     # למיגרציות עם run_migration():
     from migrations.XXX import run_migration
     run_migration(DATABASE_URL)
     
     # למיגרציות עם upgrade():
     from migrations.XXX import upgrade
     # הרצה דרך SQLAlchemy
     ```

   - אימות שהמיגרציה הצליחה (בדיקת טבלאות/עמודות)
   - המשך למיגרציה הבאה

4. דיווח מפורט על כל מיגרציה

**מיגרציות ידועות שצריך לבדוק:**

- `create_ticker_provider_symbols_table.py` - אם טבלה לא קיימת
- `create_conditions_system_tables.py` - אם טבלאות conditions לא קיימות
- `add_condition_id_to_alerts.py` - אם עמודה לא קיימת
- כל מיגרציה אחרת לפי זיהוי אוטומטי

**שינוי נדרש:**

- עדכון `scripts/run_production_migration.py` לתמוך ב-PostgreSQL במקום SQLite

### שלב 9: תיקון הגדרות production

**קובץ:** `scripts/production-update/steps/09_fix_config.py`

**פעולות:**

1. הרצת `post_sync_transformer.py` לתיקון הגדרות
2. אימות:

   - `IS_PRODUCTION = True`
   - `PORT = 5001` (קריטי!)
   - `DATABASE_URL` מצביע ל-PostgreSQL
   - `USING_SQLITE = False`

### שלב 10: בדיקת עדכונים דרושים לשרת

**קובץ:** `scripts/production-update/steps/10_check_server_updates.py` (חדש - ליצור)

**פעולות:**

1. בדיקת dependencies:

   - `requirements.txt` עודכן?
   - יש packages חדשים?
   - יש packages שצריך לעדכן?

2. בדיקת קבצים קריטיים:

   - `app.py` קיים ותקין?
   - `settings.py` תקין?
   - כל ה-routes קיימים?

3. בדיקת מבנה DB:

   - כל הטבלאות שהשרת צריך קיימות?
   - כל העמודות קיימות?

4. בדיקת קבצי UI:

   - כל קבצי HTML קיימים?
   - כל קבצי JS/CSS קיימים?

5. דיווח על עדכונים נדרשים:

   - רשימת dependencies שצריך להתקין
   - רשימת קבצים חסרים
   - אזהרות על בעיות פוטנציאליות

**פלט:**

- רשימת עדכונים נדרשים
- רשימת dependencies שצריך להתקין
- אזהרות

### שלב 11: עדכון השרת

**קובץ:** `scripts/production-update/steps/11_update_server.py` (חדש - ליצור)

**פעולות:**

1. התקנת dependencies חדשים (אם יש):
   ```bash
   cd production/Backend
   pip3 install -r requirements.txt --upgrade
   ```

2. אימות התקנת dependencies
3. בדיקת syntax של קבצים קריטיים (אופציונלי)
4. דיווח על עדכונים שבוצעו

### שלב 12: בדיקות אימות

**קובץ:** `scripts/production-update/steps/12_validate.py`

**פעולות:**

1. הרצת `verify_production_isolation.sh`
2. הרצת `verify_production.sh`
3. בדיקת מבנה DB:

   - כל הטבלאות קיימות
   - כל העמודות קיימות
   - Indexes תקינים

4. בדיקת קבצים קריטיים:

   - `app.py` קיים
   - `settings.py` תקין (PORT=5001, IS_PRODUCTION=True)
   - כל ה-routes קיימים

5. בדיקת syntax (אופציונלי)

### שלב 13: הפעלת השרת

**קובץ:** `scripts/production-update/steps/13_start_server.py` (לעדכן מ-11_start_server.py)

**פעולות:**

1. בדיקת שהשרת לא רץ (פורט 5001 פנוי)
2. הפעלת השרת:
   ```bash
   cd production
   ./start_production.sh
   # או
   ./start_server.sh --env production
   ```

3. המתנה ל-health check:

   - בדיקת `http://127.0.0.1:5001/api/health` כל שנייה
   - עד 30 שניות
   - אימות תגובה HTTP 200

4. דיווח על הפעלת השרת:

   - PID
   - זמן הפעלה
   - תוצאות health check

### שלב 14: בדיקת הרצה רציפה

**קובץ:** `scripts/production-update/steps/14_verify_server_stability.py` (חדש - ליצור)

**פעולות:**

1. המתנה 30 שניות
2. בדיקות חוזרות כל 5 שניות (6 בדיקות):

   - Health check: `GET http://127.0.0.1:5001/api/health`
   - בדיקת שהתהליך עדיין רץ (PID)
   - בדיקת לוגים לשגיאות קריטיות

3. דיווח על יציבות:

   - כל הבדיקות עברו?
   - יש שגיאות בלוגים?
   - השרת רץ יציב?

**פלט:**

- תוצאות כל הבדיקות
- סטטוס יציבות (יציב/לא יציב)
- רשימת שגיאות (אם יש)

### שלב 15: עדכון גרסה

**קובץ:** `scripts/production-update/steps/15_bump_version.py`

**פעולות:**

### שלב 16: Commit & Push

## קבצים חדשים ליצור

### 7. `scripts/production-update/steps/11_update_server.py`

**תפקיד:** עדכון השרת

**פונקציות:**

- `install_dependencies()` - התקנת dependencies
- `verify_dependencies()` - אימות התקנת dependencies
- `check_syntax()` - בדיקת syntax

### 8. `scripts/production-update/steps/14_verify_server_stability.py`

**תפקיד:** בדיקת יציבות השרת

**פונקציות:**

- `generate_stability_report()` - יצירת דוח יציבות

- עדכון ל-PostgreSQL: שימוש ב-`pg_dump` במקום SQLite backup

- הוספת steps חדשים:
  - Step 2: `02_check_server.py`
  - Step 7: `07_stop_server.py`
  - Step 8: `08_run_migrations.py`
  - Step 10: `10_check_server_updates.py`
  - Step 11: `11_update_server.py`
  - Step 14: `14_verify_server_stability.py`
- עדכון סדר השלבים
- עדכון step 11 → 13 (start_server)
