📊 סכמה כללית

┌─────────────────────────────────────────────────────────────┐ │ TikTrack Backend │ ├─────────────────────────────────────────────────────────────┤ │ 🚀 Entry Points: │ │ ├── app.py (Main Flask App) │ │ ├── run_stable.py (Production Stable) │ │ ├── run_waitress_fixed.py (Waitress WSGI) │ │ └── dev_server.py (Development with Auto-reload) │ ├─────────────────────────────────────────────────────────────┤ │ 📁 Architecture Layers: │ │ ├── 🎯 Routes Layer (API Endpoints) │ │ ├── 🔧 Services Layer (Business Logic) │ │ ├── 🗃️ Models Layer (Data Models) │ │ ├── ⚙️ Config Layer (Configuration) │ │ └── 🗄️ Database Layer (PostgreSQL + SQLAlchemy) │ └─────────────────────────────────────────────────────────────┘

📁 מבנה ארכיטקטוני מפורט

🎯 Routes Layer (API Endpoints)

Backend/routes/ ├── api/ │ ├── accounts.py # חשבונות │ ├── trades.py # טריידים │ ├── tickers.py # טיקרים │ ├── trade_plans.py # תכנוני מסחר │ ├── alerts.py # התראות │ ├── cash_flows.py # תזרימי מזומנים │ ├── notes.py # הערות │ ├── executions.py # ביצועים │ └── preferences.py # העדפות └── pages.py # דפי HTML

🔧 Services Layer (Business Logic)

Backend/services/ ├── account_service.py # לוגיקת חשבונות ├── trade_service.py # לוגיקת טריידים ├── ticker_service.py # לוגיקת טיקרים ├── trade_plan_service.py # לוגיקת תכנונים └── alert_service.py # לוגיקת התראות

🗃️ Models Layer (Data Models)

Backend/models/ ├── base.py # מודל בסיס (BaseModel) ├── account.py # מודל חשבון ├── trade.py # מודל טרייד ├── ticker.py # מודל טיקר ├── trade_plan.py # מודל תכנון מסחר ├── alert.py # מודל התראה ├── cash_flow.py # מודל תזרים מזומנים ├── note.py # מודל הערה ├── execution.py # מודל ביצוע └── swagger_models.py # מודלים ל-Swagger

⚙️ Config Layer (Configuration)

Backend/config/ ├── settings.py # הגדרות בסיסיות ├── database.py # קונפיגורציית DB ├── logging.py # הגדרות לוגים └── openapi.py # הגדרות Swagger

🔄 זרימת נתונים

Client Request → Routes → Services → Models → Database ↓ Response ← JSON ← Services ← Models ← Database
הערה: כל שכבה מטפלת בתפקיד ספציפי ומעבירה את הנתונים לשכבה הבאה, מה שמבטיח הפרדה ברורה של אחריות וקלות תחזוקה.

🛠️ טכנולוגיות עיקריות

Framework

Flask (Python) - מסגרת עבודה קלה ומהירה

Database

PostgreSQL עם SQLAlchemy ORM - ניהול נתונים יעיל

WSGI Server

Waitress (Production) / Flask Dev Server

API Documentation

Swagger/OpenAPI - תיעוד API אוטומטי

CORS

Flask-CORS - תמיכה ב-Cross-Origin Requests

Logging

Python logging עם קונפיגורציה מותאמת

🚀 שיטות הפעלה

Production Stable

python3 run_stable.py # יציב לשימוש יומיומי # Flask development server עם הגדרות יציבות # debug=False, use_reloader=False

Production Waitress

python3 run_waitress_fixed.py # WSGI server לפרודקשן # Waitress עם הגדרות מתוקנות # threads=4, connection_limit=100

Development

python3 dev_server.py # עם auto-reload ו-monitoring # מתאים לפיתוח # בודק dependencies ומתקין אם צריך

✅ מאפיינים עיקריים

  • ארכיטקטורה מודולרית - הפרדה ברורה בין שכבות
  • RESTful API - נקי ומאורגן
  • Business Logic Separation - לוגיקה עסקית בשירותים נפרדים
  • Database Abstraction - SQLAlchemy ORM
  • Error Handling - טיפול בשגיאות מרכזי
  • Logging - מערכת לוגים מפורטת
  • Health Checks - בדיקות בריאות שרת
  • CORS Support - תמיכה ב-Cross-Origin Requests
