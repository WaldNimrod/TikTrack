# Changelog - TikTrack

כל השינויים המשמעותיים במערכת TikTrack.

## [2025-08-17] - עדכון מבנה הערות וארכיטקטורת פרונטנד

### ✅ שינויים שבוצעו

#### 🔄 עדכון מבנה בסיס הנתונים - הערות
- **שינוי מבנה טבלת הערות**: מ-3 שדות נפרדים ל-2 שדות (`related_type`, `related_id`)
- **העתקת נתונים קיימים**: כל הערה משויכת לאלמנט אחד בלבד
- **יצירת אינדקסים**: לביצועים טובים יותר
- **סקריפט מיגרציה**: `update_notes_structure.py` להעתקת נתונים

#### 🔧 עדכון מודלים
- **מודל Note**: עדכון למבנה החדש עם תאימות לאחור
- **מודלים אחרים**: הסרת relationships ישנים מ-Account, Trade, TradePlan
- **תאימות לאחור**: API מחזיר `account_id`, `trade_id`, `trade_plan_id` לפי הקשר

#### 🌐 עדכון API
- **יצירת הערות**: לוגיקה לקביעת הקשר לפי עדיפות (`trade_plan` > `trade` > `account`)
- **עדכון הערות**: לוגיקה דומה לקביעת הקשר
- **תיקון endpoints**: `/api/v1/trade_plans/` (עם underscore)

#### 🎨 עדכון פרונטנד
- **ארכיטקטורה מודולרית**: קובץ JS נפרד לכל טבלה
- **פונקציות כלליות**: ריכוז ב-`grid-table.js`
- **פונקציות ייעודיות**: `accounts.js`, `alerts.js`, `notes.js`
- **תצוגה משופרת**: הצגת הקשר בהערות בצורה ברורה

#### 📊 דף בסיס נתונים
- **דף מרכזי**: `database.html` לניהול כל הנתונים
- **טבלאות פעילות**: חשבונות, טיקרים, התראות, הערות, תכנונים, תזרימים, ביצועים
- **טבלאות מוסתרות**: משתמשים ותפקידים (להמשך)
- **פונקציונליות מלאה**: הוספה, עריכה, מחיקה לכל הטבלאות

### 🔧 תיקונים טכניים

#### API Endpoints
- **תיקון trade_plans**: `/api/v1/trade_plans/` (עם underscore)
- **תיקון הערות**: מבנה חדש עם תאימות לאחור
- **תיקון חשבונות**: `/api/v1/accounts/`
- **תיקון התראות**: `/api/v1/alerts/`

#### ארכיטקטורת פרונטנד
- **ריכוז פונקציות**: `grid-table.js` לפונקציות כלליות
- **מודולריות**: קובץ JS נפרד לכל טבלה
- **חשיפה גלובלית**: כל הפונקציות זמינות דרך `window`
- **טיפול בשגיאות**: לוגים מפורטים וטיפול בשגיאות

#### CSS ותצוגה
