⚡ Agents OS — Pipeline System Guide

🚀 Quick Start (30 seconds)

# 1. Start the dashboard server (from repo root):
python3 -m http.server 8090
# → Open: http://localhost:8090/PIPELINE_DASHBOARD.html

# 2. Every gate follows the same 3-step cycle:
./pipeline_run.sh         # ← generate prompt (shows ▼▼▼ block)
# → paste ▼▼▼ block into the AI tool shown (Cursor / Codex / Claude)
# → run the AI, get result
./pipeline_run.sh pass    # ← advance gate → auto-show next

📋 All pipeline_run.sh Commands

./pipeline_run.sh                    # generate + display current gate prompt
./pipeline_run.sh pass               # advance PASS → show next gate
./pipeline_run.sh fail "reason"      # advance FAIL with reason
./pipeline_run.sh approve            # approve human gate (GATE_2, GATE_6, GATE_7)
./pipeline_run.sh status             # show current state (no prompt)
./pipeline_run.sh gate GATE_NAME     # generate prompt for a specific gate
./pipeline_run.sh store GATE file    # store AI output file into pipeline state
./pipeline_run.sh revise "notes"     # G3_5 FAIL → generate G3_PLAN revision prompt

👥 How Team Mandates Work (CURSOR_IMPLEMENTATION gate)

The Team Mandates accordion in this dashboard shows each team's mandate. The order is enforced:

1. Check "Expected Output Files" panel → Team 20 file: 🔴
2. Open Cursor Composer → paste Team 20 mandate → AI produces API verify file → 🟢
3. Open new Cursor Composer → paste Team 30 mandate (reads Team 20 file)
4. Team 30 implements → git commit
5. ./pipeline_run.sh pass   ← ONLY NOW advance CURSOR_IMPLEMENTATION
✅ Signal that Team 20 is done: The "Expected Output Files" panel shows 🟢 Team 20 API verify. Press ↺ Refresh to update.
⚠️ If Team 20 finds an API gap (endpoint missing, wrong params): STOP. Do not start Team 30. Post to Team 00 (architect) for decision before continuing.

🔄 Handling FAIL (G3_5 or GATE_4)

# After G3_5 FAIL:

./pipeline_run.sh revise "BLOCKER-1: ... BLOCKER-2: ..."
# → paste revised G3_PLAN prompt into Cursor → Team 10 fixes → ./pipeline_run.sh pass

❓ FAQ

שאלה: הרצתי Team 20. מה הטריגר הבא?

אין CLI trigger אחרי Team 20. בדוק שהקובץ _COMMUNICATION/team_20/TEAM_20_S001_P002_WP001_ALERTS_WIDGET_API_VERIFICATION_v1.0.0.md קיים (ראה "Expected Files" → 🟢). אם 🟢 — פתח Cursor חדש, הדבק מנדט Team 30.

שאלה: מתי אני לוחץ ./pipeline_run.sh pass?

רק כאשר כל ה-sub-steps של הgate הנוכחי הושלמו. עבור CURSOR_IMPLEMENTATION: Team 20 🟢 + Team 30 🟢 + git commit קיים.

שאלה: שכחתי איפה אני בפייפליין?

לחץ ↺ Refresh — ה-"Current Gate" בסיידבר מראה בדיוק איפה אתה. הaccordion "Current Gate Prompt" מראה מה צריך לעשות עכשיו.

שאלה: קיבלתי BLOCK מ-Team 190?

קרא את ה-BLOCK findings בקובץ שנוצר בתיקיית Team 190. תקן את הspec או העבר ל-Team 00 להחלטה אדריכלית. אל תמשיך הלאה ב-pipeline.
