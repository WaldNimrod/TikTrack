## ⚠️ **הערה חשובה על התפתחות המערכת**

**מערכת העדפות V2 הפכה למערכת העדפות הגנרית**

המערכת הנוכחית התפתחה ממערכת "V2" למערכת העדפות גנרית ומובילה. התהליך כלל:

1. **שלב V1** - מערכת בסיסית (הוסרה לחלוטין)
2. **שלב V2** - מערכת מתקדמת עם פרופילים מרובים
3. **שלב נוכחי** - מערכת העדפות גנרית (לשעבר V2)

**כל ההפניות ל-V2 הוסרו מהקוד והממשק** כדי לפשט את המערכת ולהפוך אותה לגנרית יותר.

## 🎯 **סקירה כללית**

מערכת העדפות של TikTrack היא מערכת מתקדמת לניהול הגדרות משתמש עם תכונות מתקדמות:

### **תכונות עיקריות:**
- ✅ **פרופילים מרובים** - כל משתמש יכול ליצור מספר פרופילי הגדרות
- ✅ **יבוא/יצוא הגדרות** - שמירה וטעינה של הגדרות
- ✅ **היסטוריית שינויים** - מעקב אחר כל השינויים
- ✅ **בדיקות תקינות** - ולידציה מתקדמת של הגדרות
- ✅ **ממשק משתמש מודרני** - עיצוב מתקדם עם אנימציות
- ✅ **תאימות לאחור** - תמיכה במערכת הישנה

### **סטטיסטיקות:**
- **50+ הגדרות** זמינות
- **8 API endpoints** פעילים
- **1,000+ שורות קוד** JavaScript
- **תמיכה מלאה** ב-RTL

## 🏗️ **ארכיטקטורה**

### **Frontend Components:**
```
trading-ui/
├── preferences.html              # ממשק משתמש ראשי
├── scripts/preferences.js        # מנוע JavaScript (1,000+ שורות)
└── styles/preferences.css        # עיצוב המערכת
```

### **Backend Components:**
```
Backend/
├── models/user_preferences.py    # מודל SQLAlchemy
├── routes/api/preferences.py     # API endpoints
├── services/preferences_service.py # לוגיקה עסקית
└── config/preferences_defaults.json # ברירות מחדל
```

### **Database Schema:**
- `user_preferences` - הגדרות משתמש
- `preference_profiles` - פרופילים מרובים
- `preference_history` - היסטוריית שינויים

## ⚡ **תכונות עיקריות**

### **1. פרופילים מרובים**
- כל משתמש יכול ליצור מספר פרופילי הגדרות
- מעבר מהיר בין פרופילים
- שמירת הגדרות נפרדות לכל פרופיל

### **2. יבוא/יצוא הגדרות**
- שמירת הגדרות לקובץ JSON
- טעינת הגדרות מקובץ
- גיבוי ושחזור הגדרות

### **3. היסטוריית שינויים**
- מעקב אחר כל השינויים
- אפשרות לחזור לגרסאות קודמות
- תאריך ושעה לכל שינוי

### **4. בדיקות תקינות**
- ולידציה מתקדמת של הגדרות
- הודעות שגיאה ברורות
- הצעות לתיקון

### **5. ממשק משתמש מודרני**
- עיצוב מתקדם עם אנימציות
- Responsive design
- תמיכה מלאה ב-RTL

## 🔌 **API Endpoints**

### **Endpoints פעילים:**
- `GET /api/preferences/` - קבלת כל ההגדרות
- `POST /api/preferences/` - עדכון הגדרות
- `GET /api/preferences/profiles/` - קבלת פרופילים
- `POST /api/preferences/profiles/` - יצירת פרופיל חדש
- `DELETE /api/preferences/profiles/{id}` - מחיקת פרופיל
- `GET /api/preferences/history/` - היסטוריית שינויים
- `POST /api/preferences/export/` - יצוא הגדרות
- `POST /api/preferences/import/` - יבוא הגדרות

## 🚀 **שימוש במערכת**

### **גישה למערכת:**
1. פתח את דף ההעדפות: `http://localhost:8080/preferences.html`
2. בחר פרופיל או צור חדש
3. עדכן הגדרות לפי הצורך
4. שמור שינויים

### **יצירת פרופיל חדש:**
1. לחץ על "פרופיל חדש"
2. הזן שם ותיאור
3. בחר הגדרות ברירת מחדל
4. שמור פרופיל

### **יבוא/יצוא הגדרות:**
1. עבור לטאב "יבוא/יצוא"
2. בחר פעולה (יבוא/יצוא)
3. בחר קובץ או הורד
4. אשר פעולה

## 🔧 **פתרון בעיות**

### **בעיות נפוצות:**

#### **הגדרות לא נשמרות:**
- בדוק חיבור לשרת
- ודא הרשאות כתיבה
- בדוק לוגים בקונסול

#### **פרופילים לא נטענים:**
- בדוק חיבור לבסיס נתונים
- ודא טבלאות קיימות
- בדוק הגדרות API

#### **יבוא/יצוא לא עובד:**
- בדוק פורמט קובץ JSON
- ודא הרשאות קריאה/כתיבה
- בדוק גודל קובץ

### **לוגים ודיבוג:**
- פתח Developer Tools (F12)
- עבור לטאב Console
- חפש הודעות שגיאה
- בדוק Network requests
