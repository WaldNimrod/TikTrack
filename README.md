# מערכת ניהול טריידים - TikTrack

מערכת מתקדמת לניהול חשבונות מסחר, תכנוני טריידים ומעקב אחר טריידים פעילים.

## תכונות עיקריות

- **ניהול חשבונות**: יצירה, עריכה וביטול חשבונות מסחר
- **בדיקת טריידים פתוחים**: מניעת ביטול/מחיקת חשבונות עם טריידים פעילים
- **תכנון טריידים**: יצירה ועריכה של תכנוני טריידים
- **מעקב טריידים**: מעקב אחר טריידים פעילים וסגורים עם ממשק משופר
- **התראות**: מערכת התראות למחירים ותנאים
- **הערות**: מערכת הערות עם קישור לאלמנטים במערכת
- **סטטיסטיקות**: דשבורד עם סטטיסטיקות כלליות
- **ממשק משתמש**: עיצוב מודרני בהשראת Apple עם כפתורים מעוצבים
- **ארכיטקטורה מודולרית**: קוד מאורגן ומתועד היטב עם פונקציות נפרדות לכל טבלה

## עדכונים אחרונים (אוגוסט 2025)

### שיפורי ממשק משתמש
- **עיצוב כפתורי שמירה**: כפתורים מעוצבים עם צבע ירוק של הלוגו ורקע לבן
- **צמצום רווחים**: אופטימיזציה של מרווחים בטופסי טריידים
- **איקונים**: הוספת איקונים לכפתורי שמירה (💾)

### שיפורי פונקציונליות
- **פיצול קוד**: העברת פונקציות טריידים לקובץ נפרד (`trades.js`)
- **עדכון מבנה בסיס נתונים**: הסרת שדה `opened_at` והחלפתו ב-`created_at`
- **תיקון סוגי טריידים**: הגבלה לשלושה סוגים בלבד (`swing`, `invest`, `pasive`)
- **שיפור טופסי עריכה**: סידור מחדש של שדות הטופס לנוחות המשתמש
- **ארגון תיעוד**: מבנה תיעוד מסודר עם קבצים חשובים מודגשים
- **מערכת התראות מתקדמת**: מחזור חיים מלא עם סטטוסים ומצבי הפעלה
- **הסרת שדה is_active**: פישוט מבנה טבלת ההתראות
- **מערכת שיוך גמישה**: שיוך התראות והערות לישויות שונות באמצעות `related_type_id` ו-`related_id`
- **מערכת אישור כפול**: חלון אזהרה שני למחיקת חשבונות עם בדיקת אובייקטים מקושרים
- **מערכת בדיקות מקיפה**: ארגון כל הבדיקות בתיקייה ייעודית עם מבנה מקצועי
- **כללי קישור טריידים-תוכניות**: הגדרת כללים ברורים לקישור בין טריידים לתוכניות עם validation אוטומטי

### תיקוני באגים
- **תיקון רשימת תוכניות**: פתרון בעיית טעינת תוכניות טרייד בטופסים
- **תיקון שמירת נתונים**: פתרון בעיות שמירה בעריכת טריידים
- **תיקון תצוגת נתונים**: הצגת שמות חשבונות וטיקרים במקום מזהה
- **תיקון סטטוסים**: עדכון כל הסטטוסים לאנגלית בבסיס הנתונים
- **תיקון פילטרים**: תיקון פילטרים גלובליים לעמוד חשבונות
- **ניקוי קבצי גיבוי**: הסרת שאריות סטטוסים בעברית מקבצי גיבוי

## התקנה והפעלה

### דרישות מערכת

- Python 3.7+
- Flask
- Flask-CORS
- SQLAlchemy
- Waitress (לייצור)

### התקנת תלויות

```bash
cd Backend
pip3 install -r requirements.txt
```

### הפעלת השרת

#### אופציה 1: הפעלה עם מנטור אוטומטי (מומלץ!)
```bash
./start_server.sh
```

#### אופציה 2: הפעלה רגילה
```bash
cd Backend
python3 run_waitress.py
```

#### עצירת השרת
```bash
./stop_server.sh
```

#### הפעלה אוטומטית אחרי אתחול המחשב
```bash
# הגדרת הפעלה אוטומטית
./setup_autostart.sh

# ביטול הפעלה אוטומטית
./disable_autostart.sh

# בדיקת סטטוס הפעלה אוטומטית
./check_autostart.sh
```

השרת יפעל על `http://127.0.0.1:8080`

## תכונות חדשות - ניהול חשבונות

### בדיקת טריידים פתוחים
המערכת בודקת אוטומטית אם יש טריידים פתוחים לפני ביטול או מחיקת חשבון:

- **ביטול חשבון**: שינוי סטטוס ל"מבוטל"
- **מחיקת חשבון**: מחיקה פיזית מהמערכת
- **הצגת אזהרה**: מודל עם טבלת הטריידים הפתוחים
- **סימבולי טיקרים**: הצגת סימבולים במקום מזהים מספריים

### סטטוסים עקביים
המערכת משתמשת בסטטוסים עקביים בעברית:
- `פתוח` - חשבון פעיל
- `סגור` - חשבון לא פעיל
- `מבוטל` - חשבון מבוטל

### דפים מרוכזים
- **דף בסיס נתונים** (`database.html`): ניהול כל הנתונים במערכת (מרכזי)
- **דף חשבונות** (`accounts.html`): ניהול חשבונות עם סטטיסטיקות
- **קוד משותף**: פונקציות משותפות בקובץ `grid-table.js`

## תכונות חדשות - מערכת הערות

### מבנה חדש
המערכת עברה למבנה חדש שבו כל הערה משויכת לאלמנט אחד בלבד:

- **מבנה ישן**: `account_id`, `trade_id`, `trade_plan_id` (3 שדות נפרדים)
- **מבנה חדש**: `related_type`, `related_id` (2 שדות בלבד)
- **עדיפות**: `trade_plan` > `trade` > `account`

## תכונות חדשות - מערכת ההתראות

### מחזור חיים מתקדם
מערכת ההתראות כוללת מחזור חיים מתקדם עם שלבים ברורים:

- **יצירה**: התראה חדשה במצב `open` עם `is_triggered = 'false'`
- **הפעלה**: עם התממשות התנאי, סטטוס הופך ל-`closed` ו-`is_triggered = 'new'`
- **קריאה**: המשתמש מסמן "קראתי", `is_triggered = 'true'`
- **ניהול**: החזרה למצב פעיל או ביטול ההתראה

### סטטוסים ומצבי הפעלה
- **Status**: `open` (פעיל), `closed` (הופעל), `cancelled` (בוטל)
- **Is_triggered**: `false` (לא הופעל), `new` (הופעל חדש), `true` (נקרא)

### סוגי התראות
- **price_alert**: התראה על מחיר
- **stop_loss**: התראה על עצירת הפסד  
- **volume_alert**: התראה על נפח מסחר
- **custom_alert**: התראה מותאמת אישית

### תיעוד מפורט
לפרטים מלאים על מערכת ההתראות, ראה: `Backend/ALERT_SYSTEM_DOCUMENTATION.md`

### תאימות לאחור
המערכת שומרת על תאימות לאחור:
- **API מחזיר**: `account_id`, `trade_id`, `trade_plan_id` (לפי הקשר)
- **פרונטנד**: עובד עם המבנה הישן
- **מבנה פנימי**: חדש ויעיל יותר

### פונקציונליות
- **הוספת הערות**: קישור לחשבון, טרייד או תכנון
- **עריכת הערות**: עדכון תוכן וקישור
- **מחיקת הערות**: מחיקה מהמערכת
- **תצוגה משופרת**: הצגת הקשר בצורה ברורה

## API Endpoints

### חשבונות
- `GET /api/v1/accounts/` - קבלת כל החשבונות
- `POST /api/v1/accounts/` - יצירת חשבון חדש
- `PUT /api/v1/accounts/<id>` - עדכון חשבון
- `DELETE /api/v1/accounts/<id>` - מחיקת חשבון

### תכנוני טריידים
- `GET /api/v1/trade_plans/` - קבלת כל תכנוני הטריידים
- `POST /api/v1/trade_plans/` - יצירת תכנון טרייד חדש
- `PUT /api/v1/trade_plans/<id>` - עדכון תכנון טרייד
- `DELETE /api/v1/trade_plans/<id>` - ביטול תכנון טרייד

### טריידים
- `GET /api/v1/trades/` - קבלת כל הטריידים
- `POST /api/v1/trades/` - יצירת טרייד חדש

### התראות
- `GET /api/v1/alerts/` - קבלת כל ההתראות
- `POST /api/v1/alerts/` - יצירת התראה חדשה
- `PUT /api/v1/alerts/<id>` - עדכון התראה
- `DELETE /api/v1/alerts/<id>` - מחיקת התראה
- `PATCH /api/v1/alerts/<id>/mark-read` - סימון כנקראה
- `PATCH /api/v1/alerts/<id>/reactivate` - החזרה למצב פעיל
- `PATCH /api/v1/alerts/<id>/cancel` - ביטול התראה

### הערות
- `GET /api/v1/notes/` - קבלת כל ההערות
- `POST /api/v1/notes/` - יצירת הערה חדשה
- `PUT /api/v1/notes/<id>` - עדכון הערה
- `DELETE /api/v1/notes/<id>` - מחיקת הערה

### סטטיסטיקות
- `GET /api/stats` - קבלת סטטיסטיקות כלליות

2. לחץ על "הוסף תכנון חדש"
3. מלא את הפרטים:
   - נכס (סימול המניה)
   - סוג השקעה (סווינג/השקעה/פאסיבי)
   - סכום השקעה
   - מחיר יעד
   - מחיר סטופ
   - סיבות להשקעה

### המרה לטרייד

1. בעמוד התכנון, לחץ על "המר לטרייד" בשורה הרצויה

1. פתח את עמוד המעקב
2. צפה בטריידים הפעילים והסגורים
3. לחץ על "הוסף טרייד חדש" ליצירת טרייד ידני

### ניהול הערות

1. פתח את דף בסיס הנתונים
2. עבור לסקשן "הערות"
3. לחץ על "הוסף הערה חדשה"
4. בחר את האלמנט לקישור (חשבון/טרייד/תכנון)
5. כתוב את תוכן ההערה

## בסיס נתונים

המערכת משתמשת ב-SQLite עם הטבלאות הבאות:

- `accounts` - חשבונות משתמשים
- `tickers` - מניות ונכסים
- `trade_plans` - תכנוני טריידים
- `trades` - טריידים פעילים וסגורים
- `executions` - ביצועי טריידים
- `alerts` - התראות (מבנה חדש: ללא is_active, עם מחזור חיים מתקדם)
- `notes` - הערות (מבנה חדש: related_type, related_id)
- `performance_snapshots` - תמונת זמן

### מבנה טבלת הערות החדש

```sql
CREATE TABLE notes (
    id INTEGER PRIMARY KEY,
    content VARCHAR(1000) NOT NULL,
    attachment VARCHAR(500),
    created_at DATETIME DEFAULT (CURRENT_TIMESTAMP),
    related_type VARCHAR(20),  -- 'account', 'trade', 'trade_plan'
    related_id INTEGER
);
```

### מבנה טבלת ההתראות החדש

```sql
CREATE TABLE alerts (
    id INTEGER PRIMARY KEY,
    account_id INTEGER,
    ticker_id INTEGER,
    type VARCHAR(50) NOT NULL,           -- 'price_alert', 'stop_loss', 'volume_alert'
    status VARCHAR(20) DEFAULT 'open',   -- 'open', 'closed', 'cancelled'
    condition VARCHAR(500) NOT NULL,     -- התנאי להפעלת ההתראה
    message VARCHAR(500),                -- הודעה למשתמש
    triggered_at DATETIME,               -- תאריך הפעלת ההתראה
    is_triggered VARCHAR(20) DEFAULT 'false',  -- 'false', 'new', 'true'
    created_at DATETIME DEFAULT (CURRENT_TIMESTAMP)
);
```

## ארכיטקטורת הפרונטנד

### מבנה קבצים
- **`database.html`**: דף מרכזי לניהול כל הנתונים
- **`accounts.js`**: פונקציות ייעודיות לחשבונות
- **`alerts.js`**: פונקציות ייעודיות להתראות (מערכת מתקדמת)
- **`notes.js`**: פונקציות ייעודיות להערות
- **`grid-table.js`**: פונקציות כלליות לכל הטבלאות

### עקרונות
- **קובץ JS נפרד לכל טבלה**: ארכיטקטורה מודולרית
- **פונקציות כלליות ב-`grid-table.js`**: מניעת כפילות
- **חשיפה גלובלית**: כל הפונקציות זמינות דרך `window`
- **טיפול בשגיאות**: לוגים מפורטים וטיפול בשגיאות
