# שיפורי יציבות השרת - TikTrack
# Server Stability Improvements

## 🎯 מטרה
לפתור את בעיית השרת שנופל כל חצי שעה וליצור מערכת יציבה יותר.

## ✅ שיפורים שבוצעו

### 1. מערכת מנטור אוטומטי
- **קובץ**: `Backend/monitor_server.py`
- **תפקיד**: מנטר את השרת ומפעיל אותו מחדש אם הוא נופל
- **תכונות**:
  - בדיקת בריאות כל 30 שניות
  - הפעלה מחדש אוטומטית
  - הגבלת ניסיונות (10 ניסיונות)
  - לוגים מפורטים

### 2. שרת Waitress
- **קובץ**: `Backend/run_waitress.py`
- **תפקיד**: שרת יציב יותר מ-Flask development server
- **תכונות**:
  - תמיכה ב-multiple threads
  - connection pooling
  - לא "נרדם" כמו Flask
  - הגדרות יציבות

### 3. Endpoint בריאות
- **קובץ**: `Backend/app.py` (שורה ~374)
- **תפקיד**: `/api/health` לבדיקת בריאות השרת
- **תכונות**:
  - בדיקת חיבור לבסיס הנתונים
  - timestamp של הבדיקה
  - תשובה מהירה

### 4. סקריפטי הפעלה משופרים
- **קבצים**:
  - `start_server.sh` - הפעלה מהתיקייה הראשית
  - `stop_server.sh` - עצירת השרת
  - `Backend/run_monitored.sh` - הפעלה עם מנטור
  - `setup_autostart.sh` - הגדרת הפעלה אוטומטית
  - `disable_autostart.sh` - ביטול הפעלה אוטומטית
  - `check_autostart.sh` - בדיקת סטטוס הפעלה אוטומטית
- **תכונות**:
  - שימוש ב-caffeinate למניעת שינה
  - הודעות ברורות
  - טיפול בשגיאות
  - הפעלה אוטומטית אחרי אתחול המחשב

### 5. לוגים מפורטים
- **קובץ**: `Backend/server_detailed.log`
- **תכונות**:
  - לוגים מפורטים של כל פעולה
  - timestamp לכל אירוע
  - רמות לוג שונות (INFO, ERROR, WARNING)

### 6. הגדרות SQLite משופרות
- **קובץ**: `Backend/app.py` (פונקציה `get_db_connection`)
- **שיפורים**:
  - `timeout=30.0` - timeout ארוך יותר
  - `check_same_thread=False` - תמיכה ב-multiple threads
  - `PRAGMA journal_mode=WAL` - ביצועים טובים יותר
  - `PRAGMA foreign_keys=ON` - אכיפת foreign keys

## 🚀 איך להפעיל את השרת

### הפעלה יומית (מומלץ!)
```bash
./start_server.sh
```

### עצירה
```bash
./stop_server.sh
```

### הפעלה אוטומטית אחרי אתחול המחשב
```bash
# הגדרת הפעלה אוטומטית
./setup_autostart.sh

# ביטול הפעלה אוטומטית
./disable_autostart.sh

# בדיקת סטטוס הפעלה אוטומטית
./check_autostart.sh
```

### בדיקת בריאות
```bash
curl http://127.0.0.1:8080/api/health
```

## 📊 מעקב אחר השרת

### צפייה בלוגים
```bash
# לוגים מפורטים
tail -f Backend/server_detailed.log

# לוגים רגילים
tail -f Backend/server.log
```

### בדיקת תהליכים
```bash
# בדיקה איזה תהליכים רצים על הפורט
lsof -i :8080

# בדיקת תהליכי Python
ps aux | grep python
```

## 🔧 פתרון בעיות

### אם השרת לא עולה:
1. בדוק שהפורט 8080 פנוי: `lsof -i :8080`
2. סגור תהליכים ישנים: `./stop_server.sh`
3. בדוק את הלוגים: `tail -f Backend/server_detailed.log`

### אם השרת נופל:
1. המנטור יפעיל אותו מחדש אוטומטית
2. בדוק את הלוגים לזיהוי הבעיה
3. אם יש יותר מ-10 נפילות, בדוק את הקוד

## 📝 הערות חשובות

- **תמיד השתמש ב-`./start_server.sh`** - זה הפתרון היציב ביותר
- **השרת רץ על פורט 8080** - לא לשנות!
- **המנטור יפעיל את השרת מחדש** אם הוא נופל
- **שמור על הטרמינל פתוח** כדי לראות הודעות

## 🎉 תוצאות צפויות

- **יציבות גבוהה יותר** - השרת לא אמור ליפול כל חצי שעה
- **הפעלה מחדש אוטומטית** - אם השרת נופל, הוא יעלה מחדש
- **מעקב טוב יותר** - לוגים מפורטים לכל בעיה
- **שימוש קל יותר** - פקודה אחת להפעלה ועצירה
