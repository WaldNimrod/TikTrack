# נוהל עבודה בסביבת Gemini Code Assist

## 1. מטרה

נוהל זה מגדיר את פרוטוקול העבודה המחייב עבור כל סוכן AI הפועל בסביבת Gemini Code Assist, ובפרט עבור צוות 101. מטרתו להבטיח אספקה אמינה של תוצרים מורכבים, תוך התמודדות עם מגבלות טכניות של סביבת העבודה (כגון מגבלת אורך פלט).

## 2. הנוהל המחייב

### 2.1 זיהוי מגבלות וחלוקה לשלבים

כל משימה הדורשת יצירת תוכן ארוך (למשל, מסמכי LLD400, דוחות אדריכליים מפורטים) תזוהה מראש. הסוכן יחלק את המשימה באופן אוטומטי למספר שלבים לוגיים, כך שכל שלב ייצר פלט שאינו חורג ממגבלות הסביבה.

### 2.2 תקשורת ושליטה בתהליך

בתחילת תהליך רב-שלבי, הסוכן יציג למשתמש את תוכנית העבודה המלאה, כולל:
- מספר השלבים הכולל.
- תיאור קצר של התוכן שיופק בכל שלב.

בסוף כל שלב (למעט האחרון), הסוכן יעצור וימתין לקבלת פקודת אישור קבועה ופשוטה מהמשתמש:

`המשך`

הסוכן לא יתקדם לשלב הבא ללא קבלת פקודה זו.

### 2.3 ולידציה פנימית והשלמת משימה

לאחר השלמת כל שלבי יצירת התוכן, יתבצע **שלב ולידציה פנימי נוסף**.

בשלב זה, הסוכן יבצע את הפעולות הבאות:
1.  יאחד את כל חלקי התוכן שנוצרו לכדי מסמך שלם.
2.  יציג את התוצר המאוחד במלואו למשתמש.
3.  יבצע בדיקת שלמות ומבנה על התוצר המאוחד כדי לוודא שלא נפלו שגיאות בתהליך האיחוד.
4.  במידת הצורך, יבצע תיקונים קלים ויציג את הגרסה המתוקנת.

**המשימה תיחשב כמושלמת רק לאחר שהולידציה הפנימית עברה בהצלחה והוצג התוצר הסופי המלא.**
