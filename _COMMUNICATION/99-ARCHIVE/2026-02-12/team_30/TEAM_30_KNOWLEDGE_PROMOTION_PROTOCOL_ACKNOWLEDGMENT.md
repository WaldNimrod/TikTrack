### 2. Transformation Layer 🔴
- ✅ כל ה-Payloads ב-Network חייבים לעבור ב-`snake_case`
- ✅ Frontend משתמש ב-`camelCase`, Backend ב-`snake_case`
- ✅ Transformation Layer אחראי על המרה בין הפורמטים

### 3. Naming Conventions 🔴
- ✅ Entities (UI, Data Attributes, CSS, API Paths) - **רבים** (`trades`, `trading_accounts`)
- ✅ Identifiers (IDs, Query Params) - **יחיד** (`trade_id`, `trading_account_id`)
- ✅ SSOT Backend References - **יחיד** (`trade_plans` - SSOT עם Backend)

### 4. CSS Standards 🔴
- ✅ Fluid Design - ללא Media Queries, שימוש ב-`clamp()`, `min()`, `max()`
