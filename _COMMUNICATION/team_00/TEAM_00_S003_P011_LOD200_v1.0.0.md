## §2 — Program Purpose

S003-P011 addresses three distinct but sequenced improvements to the Agents_OS pipeline infrastructure:

1. **WP001 — Gate Terminology Canonicalization:** Eliminates naming drift in gate IDs, state machine, and CLI. Foundational prerequisite for WP002 — the Dashboard Copilot must be built with correct vocabulary from day 1.

2. **WP002 — Dashboard Copilot (ADR-031 Stage B):** Active guided interface overlaid on the pipeline dashboard. Previously S003-P007, now delivered here with correct terminology as its foundation.

3. **WP003 — Event-Driven Watcher + SSE Push:** Replace polling-based dashboard updates with server-sent events (SSE) for real-time gate event notification.

**Sequencing constraint:** WP001 → WP002 → WP003 (strict dependency chain). WP002 MUST NOT begin until WP001 is GATE_8 PASS.

### WP003 — Event-Driven Watcher + SSE Push

**Scope:** Replace dashboard polling with server-sent events (SSE) for real-time gate advance notification.

**Key behaviors:**
- Dashboard subscribes to `/api/events/pipeline` SSE stream
- Gate ADVANCE events pushed immediately to connected dashboard clients
- Verdict file detection via `watchdog` or equivalent filesystem watcher
- Dashboard auto-refresh on GATE_ADVANCE_PASS / GATE_ADVANCE_FAIL events
- Eliminates OBS-002 (no proactive feedback when Team 190 submits verdict) — see FLIGHT_LOG

**LOD200 detail level:** Sufficient. Full LLD400 to be authored by Team 170 at GATE_1.
