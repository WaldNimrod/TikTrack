## 2. מפרט: דשבורד נתונים (Data Dashboard)

### 2.1 ניווט

- **תפריט ראשי:** כפתור/פריט **"נתונים"** — קישור לעמוד דשבורד הנתונים.

- **routes.json:** להוסיף `"data_dashboard": "/data_dashboard.html"` (או path שנבחר) ולקשר מ־תפריט "נתונים".

### 2.2 טבלה 1 — שערים שאנחנו מחזיקים

| דרישה | תיאור |
|--------|--------|
| **תוכן** | רשימת כל זוגות השערים (from_currency, to_currency) שיש ב־`market_data.exchange_rates`. |
| **עמודות** | זוג מטבעות (למשל USD→ILS), **ערך עדכני** (conversion_rate), **זמן עדכון אחרון** (last_sync_time). |
| **מקור נתונים** | GET /api/v1/reference/exchange-rates — כבר מחזיר רשימה + staleness. ניתן להרחיב/להשתמש בשדות קיימים. |
| **הערה** | אם יוחלט להציג גם staleness (צבע/טקסט) — אפשר להשתמש ב־staleness מהתשובה. |

### 2.3 טבלה 2 — היסטוריה של שער מסוים

| דרישה | תיאור |
|--------|--------|
| **תוכן** | היסטוריית שערים עבור **שער נבחר** (זוג מטבעות אחד). |
| **כותרת הטבלה** | דרופדאון (dropdown) לבחירת השער — רשימת זוגות מטבעות (אותה רשימה כמו בטבלה 1 או מתוך API). |
| **עמודות** | תאריך/שעה (as_of), ערך (conversion_rate). |
| **מקור נתונים** | **חסר כרגע:** ב-DB יש רק שער נוכחי ל־(from, to) — אין טבלת היסטוריה. נדרש אחד מהבאים: (א) טבלה `market_data.exchange_rates_history` (או דומה) + לוג בסקריפט ה-sync, או (ב) endpoint חדש שמחזיר היסטוריה מספק חיצוני. |

**פעולה Backend (Team 20/60):** להחליט וליישם אספקת **היסטוריית שערים** לשער בודד (טבלה + sync, או API חיצוני) ו־endpoint לממשק (למשל GET /api/v1/reference/exchange-rates/history?from=USD&to=ILS).
