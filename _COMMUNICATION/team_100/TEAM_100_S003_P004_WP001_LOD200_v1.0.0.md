## §2 — Purpose

D33 (User Tickers) is the "הטיקרים שלי" (My Tickers) page — a personal watchlist where users track the securities they follow. The page shows the user's ticker list with live price data, display name (Iron Rule: display_name shown; raw ticker symbol NEVER exposed in UI), and full table UX (filtering, sorting, pagination).

**This package rebuilds the legacy user_tickers page in the Phoenix architecture.** The backend infrastructure (model, schema, service, router) already exists from prior work packages. The primary scope of this WP is to deliver a complete, compliant frontend page and verify the full API+UI stack.

## §4 — Scope of This Work Package

### 4.1 Frontend (Team 30 — primary deliverable)

Build the complete `user_tickers.content.html` page:

1. **Summary bar** — total tickers count, active count
2. **Filter bar** — filter by: ticker_type (STOCK/ETF/CRYPTO), status (active/inactive), sector, market_cap_group
3. **Sortable table** — all visible columns sortable (click header)
4. **Pagination** — max 50 rows/page, page size selector (10/25/50)
5. **Table columns:**
   - `display_name` (Iron Rule: shown as primary; if null, show `company_name`; raw `symbol` NEVER shown alone as primary identifier)
   - `ticker_type`
   - `current_price` + `daily_change_pct` (live, with price_source badge: EOD/INTRADAY)
   - `currency`
   - `exchange_code`
   - `status`
   - Actions: Edit display_name | Remove from watchlist
6. **Add ticker modal** — symbol lookup, optional display_name
7. **Edit display_name modal** — PATCH /me/tickers/{id} with display_name field
8. **Empty state** — "No tickers found" handled gracefully
9. **Error state** — API failure toast with retry

### 4.3 API Contract (authoritative — Team 20 must implement exactly)

#### `GET /me/tickers` (extended)
```
Query params:
  ticker_type: STOCK|ETF|CRYPTO|... (optional, multi)
  status: active|inactive (optional)
  sort_by: display_name|current_price|daily_change_pct|ticker_type|status (default: display_name)
  sort_dir: asc|desc (default: asc)
  page: int (default: 1)
  page_size: int (default: 25, max: 50)

Response:
  {
    "data": [TickerResponse],
    "total": int,
    "page": int,
    "page_size": int
  }
```

#### `PATCH /me/tickers/{ticker_id}` (new)
```
Body: { "display_name": str|null (max 100 chars) }
Response: TickerResponse
```

## §5 — Iron Rules (non-negotiable — enforced at GATE_2 Phase 2.3 + GATE_4)

| Rule ID | Rule | Enforcement |
|---------|------|-------------|
| D33-IR-01 | **Display Lock:** `display_name` is the primary identifier in all UI. If `display_name` is null, show `company_name`. Raw `symbol` must NEVER appear as the primary visible identifier. | GATE_2 Phase 2.3 + GATE_4 visual inspection |
| D33-IR-02 | **collapsible-container:** Page uses tt-container > tt-section > tt-section-row structure | GATE_4 |
| D33-IR-03 | **Pagination max 50:** `page_size` hard-capped at 50 rows; no unbounded loads | GATE_2 Phase 2.3 |
| D33-IR-04 | **All columns sortable:** Every visible column in the table must have a sort trigger | GATE_2 Phase 2.3 |
| D33-IR-05 | **maskedLog:** All server-side logging uses `maskedLog` — no raw user data in logs | GATE_4 |
| D33-IR-06 | **Error state:** "No tickers found" rendered gracefully; API errors show toast | GATE_4 |
| D33-IR-07 | **Live price display:** `current_price` + `daily_change_pct` + `price_source` badge shown per row | GATE_4 |
