## §7 — `fail` and `pass` Command Handlers (D-01, D-06)

The CLI command handlers in `pipeline.py` will be updated to enforce stricter state management.

**`fail` Handler Pseudocode:**
1. Parse CLI arguments: `reason`, `--finding-type`, `--fcp-level`.
2. Validate `finding_type` against the FCP ENUM. If invalid, abort.
3. Load `PipelineState`.
4. Set `state.last_blocking_findings = reason`.
5. Set `state.last_blocking_gate = state.current_gate`.
6. Increment `state.remediation_cycle_count`.
7. Set `state.finding_type` and `state.fcp_level`.
8. Determine return route using `FAIL_ROUTING` table.
9. Set `state.return_target_team`.
10. Set `state.current_gate` and `state.current_phase` to the new routed values.
11. Save `PipelineState`.

**`pass` Handler Pseudocode:**
1. Parse optional `[GATE_ID]` argument from CLI.
2. Load `PipelineState`.
3. If `GATE_ID` was provided, assert `GATE_ID == state.current_gate`. If not, abort with error.
4. If `state.last_blocking_gate == state.current_gate`, prompt user for confirmation `[y/n]`.
5. Clear `state.last_blocking_findings`, `state.last_blocking_gate`.
6. Find index of `state.current_gate` in `GATE_SEQUENCE`.
7. Set `state.current_gate` to the next gate in the sequence.
8. Reset `state.current_phase` to the first phase of the new gate (e.g., "3.1").
9. Save `PipelineState`.

### §17.2 — Migration Contract (Mandatory Runtime Behavior)

`PipelineState` migration MUST run as an after-validator and persist migrated state:

```python
@model_validator(mode="after")
def _run_migration(self) -> "PipelineState":
    original_gate = self.current_gate
    migration = _MIGRATION_TABLE.get(original_gate)
    if not migration:
        return self

    self.current_gate = migration.new_gate
    self.current_phase = migration.new_phase
    if migration.set_human_pending:
        self.gate_state = "HUMAN_PENDING"
    if migration.increment_remediation:
        self.remediation_cycle_count += 1

    print(
        f"INFO: Auto-migrated {self.work_package_id}: "
        f"{original_gate} -> {self.current_gate}/{self.current_phase}"
    )
    _append_pipeline_event(
        event_type="STATE_MIGRATION",
        description=f"{original_gate} -> {self.current_gate}/{self.current_phase}",
        wp_id=self.work_package_id,
        domain=self.project_domain,
    )
    self.save()  # persist post-migration state
    return self
```

### §17.3 — Complete `_resolve_phase_owner` Contract

```python
def _resolve_phase_owner(state: PipelineState, gate: str, phase: str) -> str | list[str]:
    domain = (state.project_domain or "").lower()
    track = state.process_variant or "TRACK_FOCUSED"
    gate_table = _DOMAIN_PHASE_ROUTING.get(gate, {})
    phase_table = gate_table.get(phase, {})

    # first by track, then by domain, then fallback key
    if track in phase_table:
        owner = phase_table[track]
    elif domain in phase_table:
        owner = phase_table[domain]
    else:
        owner = phase_table.get("default")

    if owner == "lod200_author_team":
        return state.lod200_author_team
    return owner
```
