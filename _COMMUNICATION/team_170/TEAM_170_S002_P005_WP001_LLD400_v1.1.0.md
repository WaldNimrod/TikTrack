## §3 Endpoint Contract (CLI / Interface)

**Domain:** AGENTS_OS has no HTTP API. Contracts are CLI commands and file schemas.

### 3.1 CLI: pipeline_run.sh

| Command | Method | Purpose |
|---------|--------|---------|
| `./pipeline_run.sh new <PROGRAM_ID>` | CREATE state | Activate new program at GATE_0; creates `pipeline_state_agentsos.json` |
| `./pipeline_run.sh --domain agents_os pass` | UPDATE state | Advance current gate to PASS |
| `./pipeline_run.sh --domain agents_os fail "reason"` | UPDATE state | Advance current gate to FAIL |
| `./pipeline_run.sh --domain agents_os approve` | Human approval | Approve GATE_2 / GATE_6 / GATE_7 |
| `./pipeline_run.sh --domain agents_os status` | READ state | Display pipeline status |
| `./pipeline_run.sh --domain agents_os gate GATE_X` | Generate prompt | Override: generate prompt for named gate |

**`new` command (R4 — required):**
- **Request:** `./pipeline_run.sh new S002-P005` — `PROGRAM_ID` format `S\d{3}-P\d{3}`
- **Response:** Creates state file; prints `✅ Program activated: {PROGRAM_ID}-WP001 @ GATE_0`
- **Overwrite:** If state file exists → prompt confirmation; do not overwrite silently

### 3.2 Python Module: state_reader

| Entry Point | Purpose |
|-------------|---------|
| `python3 -m agents_os_v2.observers.state_reader` | Build STATE_SNAPSHOT.json |

**Output schema:** `_COMMUNICATION/agents_os/STATE_SNAPSHOT.json`

### 3.3 STATE_SNAPSHOT.json Response Schema

```json
{
  "schema_version": "2.0.0",
  "produced_at_iso": "<ISO8601>",
  "read_only": true,
  "governance": {
    "wsm_path": "<relative_path>",
    "active_stage": "S{NNN}",
    "active_program": "<program_id>",
    "ssm_path": "<relative_path>",
    "wsm_found": true,
    "master_index_path": "00_MASTER_INDEX.md"
  },
  "consistency_check": {
    "gate_integrity": {
      "status": "PASS|FAIL",
      "checked_gates": ["GATE_0", "G3_PLAN", "G3_6_MANDATES", "GATE_8"],
      "missing_artifacts": ["<path>"],
      "severity": "CRITICAL"
    },
    "wsm_stage_watch": {
      "status": "MATCH|MISMATCH|UNKNOWN",
      "pipeline_stage": "S{NNN}",
      "wsm_stage": "S{NNN}",
      "severity": "WARNING"
    },
    "overall_status": "PASS|WARN"
  }
}
```

**R1:** If required field absent in WSM → `read_governance_state()` raises `StateReadError` (never silent None).

## §5 UI Structural Contract

**Page:** `agents_os/ui/PIPELINE_ROADMAP.html` — Roadmap & Gate History

### 5.1 Consistency Banners (R2.3)

| Condition | Banner | Color | DOM Anchor |
|-----------|--------|-------|------------|
| `gate_integrity.status = FAIL` | ⛔ Gate Integrity Violation — artifacts missing. Run: `python3 -m agents_os_v2.observers.state_reader` | Red | `data-testid="gate-integrity-banner"` or `id="gate-integrity-banner"` |
| `wsm_stage_watch.status = MISMATCH` | ⚠️ WSM stage (`{wsm_stage}`) does not match pipeline stage (`{pipeline_stage}`) — verify WSM is current | Amber | `data-testid="wsm-stage-banner"` or `id="wsm-stage-banner"` |
| `overall_status = PASS` | No banner | — | Banner container hidden |

### 5.2 DOM Anchors

| Element | Anchor | Purpose |
|---------|--------|---------|
| Gate integrity banner | `data-testid="gate-integrity-banner"` | Red banner container |
| WSM stage banner | `data-testid="wsm-stage-banner"` | Amber banner container |
| Conflict warnings container | `id="conflict-warnings"` | Existing — may host banners |
| Domain stats | `id="stat-tiktrack"`, `id="stat-agentsos"`, `id="stat-total"` | Sidebar stat cards |
| Roadmap tree | `id="roadmap-tree"` | Main content |
| Refresh button | `id="refresh-btn"` | Load STATE_SNAPSHOT |

### 5.3 State Shape (Client)

`loadAll()` fetches `STATE_SNAPSHOT.json` (or equivalent path). Renders `consistency_check.gate_integrity` and `consistency_check.wsm_stage_watch` into banners per §5.1. Banners auto-clear when condition resolves (next refresh).
