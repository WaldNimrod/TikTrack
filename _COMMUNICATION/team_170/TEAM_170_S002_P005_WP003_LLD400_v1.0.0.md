## §2 Endpoint Contract

**Domain:** AGENTS_OS has no HTTP API. Contracts are CLI commands, file fetches (static JSON), and Python module entry points.

### 2.1 CLI: pipeline_run.sh

| Command | Method | Purpose |
|---------|--------|---------|
| `./pipeline_run.sh --domain agents_os pass` | UPDATE state | Advance current gate to PASS |
| `./pipeline_run.sh --domain agents_os fail "reason"` | UPDATE state | Advance current gate to FAIL |
| `./pipeline_run.sh --domain agents_os pass_with_actions "ACTION-1\|ACTION-2"` | UPDATE state | Mark gate PASS_WITH_ACTION; set pending_actions (WP002) |
| `./pipeline_run.sh --domain agents_os actions_clear` | UPDATE state | Resolve pending actions; advance gate (WP002) |
| `./pipeline_run.sh --domain agents_os override "reason"` | UPDATE state | Advance gate with override_reason logged (WP002) |
| `./pipeline_run.sh --domain agents_os insist` | UPDATE state | Stay at gate; generate correction prompt (WP002) |
| `./pipeline_run.sh --domain agents_os status` | READ state | Display pipeline status |

**Note:** `new` and `sync` subcommands are not implemented in the current `pipeline_run.sh` surface and are removed from this contract. State initialization is handled by `state.py` directly.

**Invariant (CS-02):** After any gate transition, a gate ID MUST NOT appear in both `gates_completed` and `gates_failed` simultaneously.

### 2.2 Static JSON Fetch (JS)

| Endpoint | Method | Request | Response |
|----------|--------|---------|----------|
| `_COMMUNICATION/agents_os/pipeline_state_agentsos.json` | GET | — | Pipeline state (agents_os domain) |
| `_COMMUNICATION/agents_os/pipeline_state_tiktrack.json` | GET | — | Pipeline state (tiktrack domain) |
| `_COMMUNICATION/agents_os/STATE_SNAPSHOT.json` | GET | — | Observer read-only snapshot |

**Response schema (pipeline_state_*.json):** See §3 DB Contract.

**Error behavior (CS-03):** On fetch failure → MUST render `PRIMARY_STATE_READ_FAILED` error panel; NO fallback to legacy or alternate source.

### 2.3 Python Module: state_reader

| Entry Point | Purpose |
|-------------|---------|
| `python3 -m agents_os_v2.observers.state_reader` | Build STATE_SNAPSHOT.json |

**Output path:** `_COMMUNICATION/agents_os/STATE_SNAPSHOT.json`

## §3 DB Contract

**Domain:** AGENTS_OS has no database. Data sources are JSON and markdown files.

### 3.1 Files Read

| Path | Purpose | Columns / Fields |
|------|---------|------------------|
| `_COMMUNICATION/agents_os/pipeline_state_agentsos.json` | Pipeline state (agents_os) | work_package_id, current_gate, gates_completed, gates_failed, project_domain, spec_brief, lld400_content, work_plan, mandates, last_updated, gate_state, pending_actions, phase8_content |
| `_COMMUNICATION/agents_os/pipeline_state_tiktrack.json` | Pipeline state (tiktrack) | Same schema |
| `_COMMUNICATION/agents_os/STATE_SNAPSHOT.json` | Observer output (read-only) | produced_at_iso, governance, consistency_check, pipeline.domains.* |
| `documentation/docs-governance/01-FOUNDATIONS/PHOENIX_MASTER_WSM_v1.0.0.md` | WSM (governance SSOT) | active_work_package_id, active_stage_id, current_gate, active_project_domain |

### 3.2 Files Written

| Path | Purpose | Write trigger |
|------|---------|---------------|
| `_COMMUNICATION/agents_os/pipeline_state_agentsos.json` | Pipeline state (agents_os) | pipeline_run.sh pass/fail/pass_with_actions/actions_clear/override; state.py save() |
| `_COMMUNICATION/agents_os/pipeline_state_tiktrack.json` | Pipeline state (tiktrack) | Same |
| `_COMMUNICATION/agents_os/STATE_SNAPSHOT.json` | Observer output | state_reader run |

### 3.3 pipeline_state JSON Schema

| Field | Type | Notes |
|-------|------|-------|
| work_package_id | string | e.g. `S002-P005-WP003`; `"NONE"` or `""` = inactive (CS-04) |
| project_domain | string | `agents_os` \| `tiktrack` |
| stage_id | string | e.g. `S002` |
| spec_brief | string | Populated at GATE_0 |
| current_gate | string | Gate ID; `"NONE"` or `"COMPLETE"` = inactive |
| gates_completed | string[] | List of passed gate IDs |
| gates_failed | string[] | List of failed gate IDs |
| lld400_content | string | GATE_1 artifact |
| work_plan | string | G3_PLAN artifact |
| mandates | string | G3_6_MANDATES artifact |
| last_updated | string | ISO8601 |
| gate_state | string \| null | `PASS_WITH_ACTION` \| `OVERRIDE` \| null |
| pending_actions | string[] | PWA action items |
| phase8_content | string | GATE_8 Phase 2 artifact |

**CS-04 sentinel rule:** `work_package_id = "NONE"` or `""`, or `current_gate = "NONE"` or `"COMPLETE"` → `is_active = false` in all detection paths.

### 3.4 STATE_SNAPSHOT.json Response Schema

```json
{
  "schema_version": "2.0.0",
  "produced_at_iso": "<ISO8601>",
  "read_only": true,
  "governance": {
    "wsm_path": "<path>",
    "active_stage": "S{NNN}",
    "active_program": "<program_id>",
    "active_work_package_id": "<wp_id or N/A>"
  },
  "pipeline": {
    "domains": {
      "agents_os": { "work_package_id": "…", "current_gate": "…" },
      "tiktrack": { "work_package_id": "…", "current_gate": "…" }
    }
  }
}
```

**CS-08:** Dashboard health panel MUST show `snapshot_age_seconds` derived from `produced_at_iso`. Warning (yellow) when age > 3600s; critical (red) when age > 86400s.
