## §2 Endpoint Contract

### 2.1 D39 — User Preferences

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/v1/me/preferences` | Bearer JWT | Return all 23 preference fields for current user |
| PATCH | `/api/v1/me/preferences` | Bearer JWT | Partial update; merge into `settings` JSONB + direct columns |

**GET /api/v1/me/preferences — Response 200**

```json
{
  "language": "en",
  "timezone": "UTC",
  "primary_currency": "USD",
  "date_format": "YYYY-MM-DD",
  "default_trading_account": null,
  "default_order_type": "LIMIT",
  "default_time_in_force": "DAY",
  "default_commission_unit": "%",
  "pl_method": "FIFO",
  "default_price_type": "CLOSE",
  "email_notifications_enabled": false,
  "alert_trigger_email": true,
  "weekly_summary_email": false,
  "price_alert_threshold_pct": 5.0,
  "default_status_filter": "active",
  "rows_per_page": 25,
  "default_sort_column": "updated_at",
  "api_rate_limit": 1000,
  "api_key_count": 0,
  "high_contrast_mode": false,
  "font_size": "medium",
  "rtl_mode": false,
  "decimal_separator": "."
}
```

**PATCH /api/v1/me/preferences — Request Body**

Any subset of the 23 fields. Keys map as follows:
- `timezone`, `language` → direct column update on `user_data.users`
- All others → JSONB merge into `settings`

```json
{
  "timezone": "America/New_York",
  "rows_per_page": 50
}
```

**PATCH — Response 200:** Same shape as GET response with merged values.

**Error responses:** 401 (unauthenticated), 422 (validation error on field value).

{

### 2.3 D41 — User Management (Admin-only)

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/api/v1/admin/users` | Admin | Paginated user list with filters |
| PATCH | `/api/v1/admin/users/{id}/status` | Admin | Set is_active |
| PATCH | `/api/v1/admin/users/{id}/role` | Admin | Set role (USER or ADMIN only) |

**GET /api/v1/admin/users — Query Params**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| page | int | 1 | 1-based page |
| rows_per_page | int | 25 | 10/25/50/100 (from D39 preference or default 25) |
| search | string | — | Search email, first_name, last_name, display_name |
| role | string | — | USER \| ADMIN \| SUPERADMIN |
| is_active | bool | — | true \| false |

**GET — Response 200**

```json
{
  "items": [
    {
      "id": "uuid",
      "email": "user@example.com",
      "display_name": "John Doe",
      "role": "USER",
      "is_active": true,
      "phone_number": "+1234567890",
      "last_login_at": "2026-03-18T10:00:00Z",
      "created_at": "2026-01-01T00:00:00Z"
    }
  ],
  "total": 42,
  "page": 1,
  "rows_per_page": 25,
  "pages": 2
}
```

**display_name:** `first_name + " " + last_name`; if both null, fallback to `username`.

**PATCH /api/v1/admin/users/{id}/status — Request Body**

```json
{
  "is_active": false
}
```

**PATCH /api/v1/admin/users/{id}/role — Request Body**

```json
{
  "role": "ADMIN"
}
```

Allowed values: `USER`, `ADMIN`. `SUPERADMIN` transitions blocked (403).

**Self-protection (API):** 403 if target user id = current user id for status or role change.

### 3.2 Schema Changes (WP001 scope)

**Migration 1 — user_data.users**

```sql
ALTER TABLE user_data.users
ADD COLUMN IF NOT EXISTS settings JSONB NOT NULL DEFAULT '{}'::jsonb;
```

**Migration 2 — admin_data.feature_flags**

```sql
CREATE TABLE IF NOT EXISTS admin_data.feature_flags (
    key VARCHAR(100) PRIMARY KEY,
    value_bool BOOLEAN,
    value_text VARCHAR(500),
    description TEXT,
    updated_by UUID REFERENCES user_data.users(id) ON DELETE SET NULL,
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**Seed — admin_data.feature_flags (3 rows)**

```sql
INSERT INTO admin_data.feature_flags (key, value_bool, description) VALUES
  ('maintenance_mode', false, 'Put system into read-only maintenance mode'),
  ('smtp_enabled', false, 'Enable outbound email notifications'),
  ('new_user_registration_enabled', true, 'Allow new user self-registration')
ON CONFLICT (key) DO NOTHING;
```

**Seed — market_data.system_settings (3 new keys)**

```sql
INSERT INTO market_data.system_settings (key, value, value_type, description) VALUES
  ('trading_hours_start', '09:30', 'string', 'Market trading hours start (HH:MM local)'),
  ('trading_hours_end', '16:00', 'string', 'Market trading hours end (HH:MM local)'),
  ('trading_timezone', 'America/New_York', 'string', 'Timezone for trading hours evaluation')
ON CONFLICT (key) DO NOTHING;
```

### 3.3 Query Patterns

| Operation | Table | Pattern |
|-----------|-------|---------|
| GET preferences | user_data.users | SELECT id, timezone, language, settings WHERE id = :current_user_id |
| PATCH preferences | user_data.users | UPDATE users SET timezone=?, language=?, settings = settings \|\| ?::jsonb WHERE id = ? |
| GET users list | user_data.users | SELECT with filters, pagination, ORDER BY created_at DESC |
| PATCH user status | user_data.users | UPDATE users SET is_active=? WHERE id=? AND id != :current_user_id |
| PATCH user role | user_data.users | UPDATE users SET role=? WHERE id=? AND id != :current_user_id AND role != 'SUPERADMIN' |
