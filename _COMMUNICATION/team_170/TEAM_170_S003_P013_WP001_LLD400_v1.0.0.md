## 2. Endpoint Contract

### 2.1 Resource and path

- **Logical resource (brief):** Current authenticated user’s linked rows in `user_data.user_tickers`, enriched with `market_data.tickers` (and price fields per existing SSOT).
- **Canonical HTTP surface (implementation):** `GET /api/v1/me/tickers`  
  - Router: `me_tickers` (`prefix=/me`, route `GET /tickers`), mounted under `settings.api_v1_prefix` (`/api/v1`).
- **Note:** There is **no** separate `GET /api/v1/user_tickers` route in the active FastAPI map. This LLD treats **“GET /api/v1/user_tickers” in stakeholder language** as referring to **this same list response** (user’s ticker list). Any future alias path is **out of scope** for this WP unless Team 00 mandates it.

### 2.2 HTTP method and request

| Item | Specification |
|------|----------------|
| Method | `GET` |
| Path | `/api/v1/me/tickers` |
| Auth | Required — same as existing “my tickers” list (Bearer JWT / session contract unchanged). |
| Query parameters | None (unchanged). |
| Request body | None. |

### 2.3 Response schema (normative)

**Envelope:** `TickerListResponse` (existing Pydantic model)

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `data` | `TickerResponse[]` | Yes | Ordered list of tickers for the current user. |
| `total` | `integer` | Yes | `len(data)` (existing behavior). |

**Each `TickerResponse` element MUST include** (among existing fields):

| Field | JSON type | Required | Constraints |
|-------|-----------|----------|---------------|
| `display_name` | string or `null` | Yes (key present) | Sourced from `user_data.user_tickers.display_name`. Max length **100** when non-null. Whitespace-only SHALL be normalized to `null` at persistence boundaries outside this WP; **response** reflects DB as read (existing service behavior). |

All other `TickerResponse` fields remain as today (ULID `id`, `symbol`, prices, `status`, `company_name`, etc.). **No new response fields** beyond ensuring `display_name` is present and populated per DB for this WP.

### 2.4 Errors

Unchanged from existing `GET /me/tickers`: `401` unauthenticated; `500` / typed app errors per `HTTPExceptionWithCode` SSOT.

### 2.5 Explicitly out of scope (D33 read-only)

- **D33 UI** MUST NOT offer create/update/delete of `display_name` (no modal, no row action, no inline editor that PATCHes `display_name`).
- **`PATCH /api/v1/me/tickers/{ticker_id}`** may remain in the codebase for other clients or future pages; **this WP does not require removing the route**. It **forbids** D33 from invoking it for `display_name` after this change.

## 3. DB Contract

### 3.1 Tables

| Schema.table | Role |
|--------------|------|
| `user_data.user_tickers` | Junction: `user_id`, `ticker_id`, soft delete `deleted_at`, plus **`display_name` VARCHAR(100) NULL** (already deployed — **no DDL change** for this WP). |
| `market_data.tickers` | Join target for ticker attributes; **read** only for list assembly. |
| `market_data.exchanges` (or equivalent join used today) | **Read** for exchange metadata in list rows (unchanged). |
| Price-related tables | **Read** via existing `_get_price_with_fallback` path (unchanged). |

### 3.2 Columns read (list endpoint)

| Table | Columns read (minimum for this WP) |
|-------|-------------------------------------|
| `user_data.user_tickers` | `ticker_id`, `user_id`, `deleted_at`, **`display_name`** |
| `market_data.tickers` | All fields required to build `TickerResponse` today (unchanged). |

### 3.3 Columns written

**None** for `GET /api/v1/me/tickers`. This WP is **read-path + D33 presentation** only.

### 3.4 Query pattern (normative intent)

- Select active `UserTicker` rows for `current_user.id` where `deleted_at IS NULL`, join `Ticker` where `Ticker.deleted_at IS NULL`, order by `Ticker.symbol` ascending (existing).
- Map each row’s `user_tickers.display_name` into `TickerResponse.display_name` (existing `_ticker_to_response` / `model_copy(update={"display_name": ...})` pattern).

### 3.5 Schema migrations

**Forbidden** unless a separate GIN/ADR mandates: **no** new columns, tables, or renames for this WP.

## 4. UI Structural Contract (D33)

**Page:** הטיקרים שלי — `user_tickers` / D33 (`/user_tickers.html` in dev routing; SSOT URL per `STAGES_PAGES_AND_MODULES_REFERENCE`).

### 4.1 Component hierarchy

1. **Page shell** — existing `tt-container` / `index-section` layout (`user_tickers.content.html`).
2. **Summary strip** — unchanged (`#totalTickers`, `#activeTickers`, staleness card).
3. **Table block** — `table#userTickersTable.phoenix-table` → `tbody#userTickersTableBody` populated by `userTickerTableInit.js`.
4. **Row model** — one `<tr>` per ticker in the current page slice; **`data-ticker-id`** = ticker ULID string (existing).

### 4.2 Column layout (normative)

| # | Column purpose | Header (HE) | Behavior |
|---|----------------|-------------|----------|
| 0 | **Ticker symbol** | סמל | Always shows `symbol` from API (primary text). Sort key: `symbol`. |
| 1 | **Display name** | שם תצוגה | If `display_name` is non-null and non-empty after trim: show that string (normal text color). If null/empty: show **`symbol`** as fallback text with **muted / grey** styling (see §4.4). Sort key: `display_name` (nulls/empty sort consistently with symbol fallback for display only). |
| 2+ | Price, source, last close, change, company, type, status, actions | Unchanged labels and behavior except **actions** (§4.3). |

**Removal:** The first column MUST NOT combine symbol and display name into a single ambiguous cell (replace current “סמל / שם תצוגה” combined behavior).

### 4.3 Actions column

- **Remove** the control that edits `display_name` (e.g. `js-action-edit` / “שנה שם תצוגה”) from D33.
- **Retain** view, note, delete (and any other non–display_name actions) per existing SSOT unless another WP removes them.

### 4.4 Styling (NULL fallback)

- Fallback text (symbol shown when `display_name` is absent) MUST use **`color: var(--color-text-muted)`** (from `phoenix-base.css`) or an **existing** Phoenix table muted utility class if one already applies that token — **prefer reuse** over new CSS classes per project CSS rules.
- **Contrast:** Muted styling MUST be visibly distinct from the primary symbol column on the same row.

### 4.5 DOM anchors (`data-testid`) — required additions

All on D33 user tickers table unless already present with same semantics:

| Element | `data-testid` |
|---------|----------------|
| Table | `d33-user-tickers-table` |
| Header row | `d33-user-tickers-thead` |
| Body | `d33-user-tickers-tbody` |
| Each data row | `d33-user-ticker-row` + `data-ticker-id="<ULID>"` (keep `data-ticker-id`) |
| Cell — symbol column | `d33-cell-symbol` |
| Cell — display name column | `d33-cell-display-name` |
| Cell — actions (if tests need scope) | `d33-cell-actions` |

### 4.6 Client fetch and state shape

- **HTTP:** Frontend continues to call **`GET /me/tickers`** (relative to API base) via `sharedServices.get` — equivalent to `GET /api/v1/me/tickers`.
- **Row state (per row, conceptual):**  
  ` { id, symbol, display_name | null, ...existingTickerResponseFields } `  
  Table init MUST read `display_name` (camelCase `displayName` tolerated only as legacy alias from adapters; **canonical API field is snake_case** per backend).

### 4.7 View modal / other D33 surfaces

- **View details modal:** MAY continue to show symbol and, when set, a distinct “שם תצוגה” line; MUST NOT imply edit. **No** PATCH from modal for `display_name`.

## 6. Acceptance Criteria

1. **`GET /api/v1/me/tickers`** returns **`display_name`** on **every** item in `data` (string or `null`), consistent with `user_data.user_tickers.display_name` for the authenticated user’s non-deleted links.
2. **OpenAPI / `TickerResponse`** documents `display_name` as today; no regression removing the field from the list response.
3. **D33 table** has **two** leading columns: **סמל** (symbol only) and **שם תצוגה** (rules in §4.2).
4. When **`display_name`** is null or empty, the **שם תצוגה** column shows the **ticker symbol** with **grey/muted** styling per §4.4.
5. When **`display_name`** is set, the **שם תצוגה** column shows that value with **normal** (non-muted) body text styling.
6. **D33** exposes **no** UI to **create, update, or clear** `display_name` (no PATCH from this page; edit affordance removed).
7. **Required `data-testid` attributes** (§4.5) are present on the D33 table, body, and symbol/display-name cells for MCP/E2E.
8. **Empty state** row `colspan` matches the **new** column count (table remains accessible and layout-correct).
9. **Sorting** remains functional; **שם תצוגה** column has a defined sort behavior (recommend: sort by effective display string = `display_name ?? symbol`).
10. **No new database migrations** are introduced by this WP; verification uses existing `VARCHAR(100) NULL` column.
