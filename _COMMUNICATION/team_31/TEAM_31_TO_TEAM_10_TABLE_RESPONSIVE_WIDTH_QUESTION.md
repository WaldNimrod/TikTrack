## 🔍 ניתוח המצב הנוכחי

### מה קיים כרגע:

1. **תיעוד רספונסיביות (`TT2_RESPONSIVE_FLUID_DESIGN.md`):**
   ```css
   .phoenix-table-wrapper {
     overflow-x: auto;
   }
   .phoenix-table {
     min-width: 800px; /* רוחב מינימלי */
     width: 100%;
   }
   ```

2. **מימוש נוכחי (`phoenix-components.css`):**
   ```css
   .phoenix-table-wrapper {
     display: block;
     width: 100%;
     overflow-x: auto;
     overflow-y: visible;
   }
   .phoenix-table {
     width: 100%;
     /* אין min-width מוגדר כרגע */
   }
   ```

3. **מפרט טבלאות (`PHOENIX_TABLES_SPECIFICATION.md`):**
   - "רוחב מלא - כל הטבלאות תופסות את כל רוחב הקונטיינר"
   - אבל לא מפרט איך להתמודד עם רוחב דינמי

### הקונפליקט:

- **Container מקסימום:** 1400px
- **דרישה:** רוחב מלא של הטבלה
- **בעיה:** אם הטבלה רחבה מדי, היא יוצרת גלילה אופקית
- **שאלה:** איך להגדיר רוחב דינמי של עמודות כך שהטבלה תתאים לרוחב הקונטיינר?

## ❓ שאלות להבהרה

### שאלה 1: אסטרטגיית רוחב טבלאות

**מהי האסטרטגיה הנכונה לטבלאות רספונסיביות?**

**אופציה A:** טבלאות עם `width: 100%` ללא `min-width` - עמודות מתאימות אוטומטית
- ✅ יתרון: תמיד מתאימות לרוחב הקונטיינר
- ❌ חסרון: עמודות עלולות להיות צרות מדי בתוכן

**אופציה B:** טבלאות עם `min-width` ו-`overflow-x: auto` - גלילה אופקית במובייל
- ✅ יתרון: שמירה על רוחב מינימלי לתוכן
- ❌ חסרון: יוצר גלילה אופקית גם במסכים גדולים

**אופציה C:** טבלאות עם עמודות דינמיות (`table-layout: auto` או `grid-template-columns`)
- ✅ יתרון: עמודות מתאימות לתוכן ולרוחב הזמין
- ❓ שאלה: איך מגדירים רוחב מינימלי/מקסימלי לכל עמודה?

**אופציה D:** שילוב - טבלאות עם `width: 100%` + עמודות עם `min-width` ו-`max-width` דינמיות
- ✅ יתרון: גמישות מקסימלית
- ❓ שאלה: איך מגדירים את הרוחבים הדינמיים?

### שאלה 2: התנהגות בגדלי מסך שונים

**מהי ההתנהגות הנדרשת?**

1. **מסך גדול (1400px+):**
   - האם הטבלה חייבת למלא את כל הרוחב?
   - האם מותר גלילה אופקית אם יש הרבה עמודות?

2. **מסך בינוני (768px - 1400px):**
   - האם הטבלה חייבת למלא את כל הרוחב?
   - האם מותר גלילה אופקית?

3. **מסך קטן (< 768px):**
   - האם מותר גלילה אופקית?
   - האם צריך להסתיר עמודות מסוימות?

### שאלה 3: רוחב עמודות דינמי

**איך מגדירים רוחב דינמי של עמודות?**

**אפשרויות:**

1. **`table-layout: auto` (ברירת מחדל):**
   - עמודות מתאימות לתוכן
   - ❓ איך מגבילים רוחב מינימלי/מקסימלי?

2. **`table-layout: fixed` + `width` לכל עמודה:**
   - שליטה מלאה על רוחב
   - ❓ איך מגדירים רוחב דינמי (אחוזים/`fr`)?

3. **CSS Grid במקום Table:**
   - גמישות מקסימלית עם `grid-template-columns`
   - ❓ האם זה מותר מבחינה סמנטית/נגישות?

4. **שילוב: `table-layout: auto` + `min-width`/`max-width` לעמודות:**
   - גמישות עם הגבלות
   - ❓ איך מגדירים את הערכים?

### שאלה 4: RTL והתנהגות

**האם יש שיקולים מיוחדים ל-RTL?**

- האם ההתנהגות זהה ל-LTR?
- האם יש שיקולים מיוחדים ל-`inset-inline-start`/`inset-inline-end`?
