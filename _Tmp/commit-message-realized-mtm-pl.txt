Add Realized P/L and MTM P/L fields to Executions table

## Overview
Added two new fields to the executions table across the entire system:
- Realized P/L: Total realized profit/loss after commissions, interest, currency rates, etc.
- MTM P/L: Total realized profit/loss Mark-to-Market

## Database Changes
- Added `realized_pl INTEGER NULL` column to executions table
- Added `mtm_pl INTEGER NULL` column to executions table
- Updated create_fresh_database.py schema

## Backend Changes
- Updated Execution model (Backend/models/execution.py) with new fields
- Updated import_orchestrator.py to save realized_pl and mtm_pl from IBKR import
- Added validation logic (Backend/services/validation_service.py):
  - Buy actions: realized_pl always NULL (disabled in UI)
  - Sell actions: realized_pl required (NOT NULL)
  - MTM P/L: optional in both buy and sell

## Frontend Changes
- Added Realized P/L and MTM P/L columns to executions table (executions.html)
- Updated updateExecutionsTableMain() to display new fields with color coding
- Added fields to add/edit modals (executions-config.js) with dynamic logic:
  - Realized P/L disabled for buy actions, required for sell actions
  - MTM P/L always optional
- Updated saveExecution() and updateExecution() to handle new fields
- Added updateRealizedPLField() function for dynamic field updates
- Added event listeners for automatic field state changes
- Updated import preview tables to display new fields

## Import Process
- IBKR connector already parses Realized P/L and MTM P/L columns
- Import orchestrator now saves these values to database
- Preview display shows the new fields in import tables

## Documentation Updates
- Updated user-data-import-system.md with field behavior details
- Updated import-user-data-guide.md with user instructions
- Updated EXECUTIONS_FLEXIBLE_ASSOCIATION.md with schema changes
- Updated EXECUTIONS_USER_GUIDE.md with field descriptions

## Field Behavior
- Realized P/L:
  - Buy: Always NULL (disabled in UI)
  - Sell: Required (NOT NULL constraint)
- MTM P/L:
  - Buy/Sell: Optional (nullable)
