⚡ Pipeline Guide

⚡ מדריך Pipeline

Mode 1Legacy — direct team work. Nimrod + Team 10 coordinate manually. No pipeline automation. For edge cases and special situations.
Mode 2Semi-automatic — AOS + dashboard (current). Pipeline manages routing. You run pipeline_run.sh commands; dashboard shows next step.
Mode 3Fully automatic (future). Pipeline engine orchestrates everything. Human involvement only at GATE_2 (intent) and GATE_7 (UX review).

FAQ

Q: I got a DOMAIN AMBIGUOUS error.
A: Add --domain to your command: ./pipeline_run.sh --domain agents_os pass

Q: I'm at GATE_1 and dashboard shows Phase 2.
A: Team 170 submitted LLD400. Mandate tab shows "▶ SEND NOW" for Team 190 (Codex).

Q: Team 190 returned BLOCK at GATE_1 Phase 2.
A: Pipeline auto-clears LLD400, resets to Phase 1. Team 170 revises and re-submits.

Q: I ran Team 20. What's next?
A: Check "Expected Output Files" → 🟢. If green, open new Cursor → paste Team 30 mandate.

Q: When do I press ./pipeline_run.sh pass?
A: Only when ALL sub-steps are done. For CURSOR_IMPLEMENTATION: Team 20 🟢 + Team 30 🟢 + git commit.

Q: I forgot where I am.
A: "Current Step Banner" (above Gate Context) shows gate, actor, next 3 steps. Or: ./pipeline_run.sh status

Q: I got a BLOCK from Team 190 (not GATE_1).
A: Read findings in Team 190 output. Fix spec or escalate to Team 00.

🔧 Troubleshooting

Dashboard shows wrong gate after command

Press ↺ Refresh. Dashboard reads pipeline_state; it updates after pipeline_run.sh runs.

Phase 2 banner after GATE_1 BLOCK

./pipeline_run.sh --domain agents_os fail "reason" — clears lld400_content. Dashboard resets to Phase 1.

store command fails with exit 1

Check: (1) file path exists, (2) GATE_ID is GATE_1, G3_PLAN, or CURSOR_IMPLEMENTATION
