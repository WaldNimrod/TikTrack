👥 How Team Mandates Work (CURSOR_IMPLEMENTATION gate)

👥 כיצד עובדים מנדטים של צוותים

The Team Mandates accordion in this dashboard shows each team's mandate. The order is enforced:

1. Check "Expected Output Files" panel → Team 20 file: 🔴
2. Open Cursor Composer → paste Team 20 mandate → AI produces API verify file → 🟢
3. Open new Cursor Composer → paste Team 30 mandate (reads Team 20 file)
4. Team 30 implements → git commit
5. ./pipeline_run.sh pass   ← ONLY NOW advance CURSOR_IMPLEMENTATION
✅ Signal that Team 20 is done: The "Expected Output Files" panel shows 🟢 Team 20 API verify. Press ↺ Refresh to update.
⚠️ If Team 20 finds an API gap (endpoint missing, wrong params): STOP. Do not start Team 30. Post to Team 00 (architect) for decision before continuing.

1. Check "Expected Output Files" panel → Team 20 file: 🔴
2. Open Cursor Composer → paste Team 20 mandate → AI produces API verify file → 🟢
3. Open new Cursor Composer → paste Team 30 mandate (reads Team 20 file)

❓ FAQ

❓ שאלות נפוצות

Q: I ran Team 20. What's the next trigger?

There is no CLI trigger after Team 20. Check that the file exists (see "Expected Files" → 🟢). If 🟢 — open a new Cursor, paste Team 30 mandate.

Q: When do I press ./pipeline_run.sh pass?

Only when all sub-steps of the current gate are done. For CURSOR_IMPLEMENTATION: Team 20 🟢 + Team 30 🟢 + git commit exists.

Q: I forgot where I am in the pipeline?

Press ↺ Refresh — "Current Gate" in the sidebar shows where you are. Or press 🔍 בדוק התקדמות (key p) for full diagnostics.

Q: Team 90 rejected the work plan (G3_5 FAIL). What to do?

Press 🔍 בדוק התקדמות — the dashboard detects REVISION mode and shows the steps. Then: ./pipeline_run.sh fail "BLOCKER-1: ...", ./pipeline_run.sh revise "...", paste ▼▼▼ into Cursor → Team 10 fixes → ./pipeline_run.sh pass.

Q: Team 90 rejected at GATE_5 (dev validation). What to do?

Press 🔍 בדוק התקדמות. The issue is in code: ./pipeline_run.sh fail "FINDING-1: ..." → send findings to Team 10 → re-run CURSOR_IMPLEMENTATION → GATE_4 → GATE_5.

Q: Auto-refresh doesn't detect state change?

The dashboard reads pipeline_state.json. If you haven't run ./pipeline_run.sh pass/fail, state won't change. Press 🔍 בדוק התקדמות for manual check.

Q: I got a BLOCK from Team 190?

Read the BLOCK findings in the file in Team 190's folder. Fix the spec or route to Team 00. Do not proceed in the pipeline.

שאלה: הרצתי Team 20. מה הטריגר הבא?

אין CLI trigger אחרי Team 20. בדוק שהקובץ _COMMUNICATION/team_20/TEAM_20_S001_P002_WP001_API_VERIFY_v1.0.0.md קיים (ראה "Expected Files" → 🟢). אם 🟢 — פתח Cursor חדש, הדבק מנדט Team 30.

שאלה: מתי אני לוחץ ./pipeline_run.sh pass?

רק כאשר כל ה-sub-steps של הgate הנוכחי הושלמו. עבור CURSOR_IMPLEMENTATION: Team 20 🟢 + Team 30 🟢 + git commit קיים.

שאלה: שכחתי איפה אני בפייפליין?

לחץ ↺ Refresh — ה-"Current Gate" בסיידבר מראה בדיוק איפה אתה. ה-accordion "Current Gate Prompt" מראה מה צריך לעשות עכשיו. לחלופין: לחץ 🔍 בדוק התקדמות (או מקש p) לאבחון מלא.

שאלה: Team 90 פסל את תוכנית העבודה (G3_5 FAIL). מה עושים?

לחץ 🔍 בדוק התקדמות — הדשבורד יזהה אוטומטית מצב REVISION ויציג את הצעדים. בקצרה:

./pipeline_run.sh fail "BLOCKER-1: ... BLOCKER-2: ..."
./pipeline_run.sh revise "BLOCKER-1: ... BLOCKER-2: ..."
# → paste ▼▼▼ block into Cursor → Team 10 fixes → ./pipeline_run.sh pass

שאלה: Team 90 פסל ב-GATE_5 (dev validation). מה עושים?

לחץ 🔍 בדוק התקדמות. הבעיה היא בקוד: ./pipeline_run.sh fail "FINDING-1: ..." → שלח findings ל-Team 10 לתיקון קוד → CURSOR_IMPLEMENTATION מחדש → GATE_4 → GATE_5.

שאלה: ה-auto-refresh לא מזהה שינוי מצב?

ה-dashboard קורא את pipeline_state.json — אם הסוכן עדיין לא סיים לכתוב את הoutput ולא רצת ./pipeline_run.sh pass/fail, המצב לא ישתנה. לחץ 🔍 בדוק התקדמות לבדיקה ידנית + הנחיות.

שאלה: קיבלתי BLOCK מ-Team 190?

קרא את ה-BLOCK findings בקובץ שנוצר בתיקיית Team 190. תקן את הspec או העבר ל-Team 00 להחלטה אדריכלית. אל תמשיך הלאה ב-pipeline.
