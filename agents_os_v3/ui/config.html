System configuration — database-backed pipeline rules

דף זה מציג את ה-seed data שמגדיר כיצד הפייפליין פועל: Routing rules (מי מטפל בכל gate), Templates (מה כותבים לאגנט), Policies (מגבלות ו-defaults). כל שינוי דורש עדכון ב-definition.yaml ו-reseed.

Routing rules — קובעות לאיזה צוות (role) יגיע כל שלב בפייפליין, לפי gate + domain + variant. ה-ORCHESTRATOR נבחר כאן לפי ה-rule עם ה-priority הנמוכה ביותר שמתאים ל-WP. Read-only.

id gate_id phase_id domain_id variant role_id priority

Templates — הוראות הפעולה שה-ORCHESTRATOR מקבל בכל gate. הפרומפט שנוצר ב-pipeline הוא template של ה-gate הנוכחי + context דינמי מ-DB (state, actor, WP). לחץ “Preview body” לקריאה. Read-only.

Policies — הגדרות שמשפיעות על התנהגות ה-pipeline בזמן ריצה: מספר מחזורי correction מקסימלי, ה-variant הברירת מחדל לפי domain, סוג gate (human/auto). מוחלות לפי scope: GLOBAL > DOMAIN > GATE. Read-only.

policy_key scope_type domain gate_id policy_value (parsed)
