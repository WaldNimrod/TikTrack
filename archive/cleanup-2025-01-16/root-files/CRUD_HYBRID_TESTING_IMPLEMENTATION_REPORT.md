## שינוי שם טבלה גנרי

- הוחלף `note_relation_types` ב-`entity_relation_types`:
  - מודלים: `Backend/models/alert.py`, `Backend/models/note.py`, `Backend/models/note_relation_type.py`
  - API: `Backend/routes/api/note_relation_types.py` → נתיב `/api/entity_relation_types`
  - רישום בלופרינטים: `Backend/routes/api/__init__.py`, `Backend/app.py`
  - יצירת DB חדש: `Backend/create_fresh_database.py` (DDL + seed)
  - DB נקי: `Backend/scripts/create_clean_database.py` (DDL, אינדקסים, seed)
  - Frontend: `trading-ui/scripts/modules/data-basic.js`, `trading-ui/scripts/db-extradata.js`, `trading-ui/scripts/crud-testing-dashboard.js`
  - Cache allowlist: `Backend/utils/response_optimizer.py`

### מיגרציה ל-DB קיים

- נוספה מיגרציה גשר: `Backend/migrations/migration_20251013_120000_entity_relation_types_bridge.json`
  - יוצרת VIEW `entity_relation_types` עם טריגרים (INSERT/UPDATE/DELETE) מול הטבלה הישנה כדי לא לשבור FKs קיימים.
  - מאפשרת מעבר הדרגתי לשם החדש.

## בדיקות CRUD (תקציר)

- Create/Update/Delete בעמודים: trades, trade_plans, executions, alerts, notes, accounts, cash_flows, tickers
  - בכל פעולה בוצע ניקוי מטמון פר-ישות דרך UnifiedCacheManager ורענון טבלאות.
  - אזהרות אלמנטים מקושרים מופעלות במחיקות ובביטולים (modal חסימה) לפי `/api/linked-items/...`.

## 🔍 מערכת ההמלצות

### **המלצות אוטומטיות לפי סוג בעיה:**

1. **Page Load failed** → בדיקת נגישות העמוד (עדיפות 1)
2. **API Load failed** → בדיקת פעילות endpoint (עדיפות 1)
3. **CREATE failed** → בדיקת נתוני דמו (עדיפות 2)
4. **Slow response** → אופטימיזציית ביצועים (עדיפות 3)

### **המלצות כלליות:**
- בעיות בעמודי משתמש → בדיקות UI מפורטות
- בעיות ביצועים → בדיקת אופטימיזציית מסד נתונים
- בעיות נרחבות → סקירה מערכתית
