## 🧪 מה אתה צריך לבדוק בממשק

### בדיקה 1: טעינת דף Executions
**מטרה:** וידוא שהדף נטען ללא שגיאות

1. פתח דפדפן: `http://localhost:8080/executions`
2. **בדוק:**
   - ✓ הדף נטען ללא שגיאות JavaScript (פתח Console)
   - ✓ הטבלה מציגה את 6 העסקאות הקיימות
   - ✓ אין הודעות שגיאה אדומות

**תוצאה צפויה:** הדף נטען תקין, טבלה מוצגת עם כל העסקאות.

---

### בדיקה 2: פתיחת מודל הוספה
**מטרה:** וידוא שהרדיו באטן מוצג ופועל

1. לחץ כפתור **"הוסף"** בראש הדף
2. **בדוק שמוצג:**
   - ✓ כותרת "סוג שיוך"
   - ✓ שני רדיו באטן: "שיוך לטיקר" ו-"שיוך לטרייד"
   - ✓ "שיוך לטיקר" נבחר כברירת מחדל ✓
   - ✓ שדה "טיקר" מוצג
   - ✓ שדה "טרייד" **מוסתר**

**תוצאה צפויה:** רדיו באטן פעיל, "שיוך לטיקר" נבחר, שדה טיקר מוצג.

---

### בדיקה 3: החלפת סוג שיוך (טיקר → טרייד)
**מטרה:** וידוא שהשדות מתחלפים

1. במודל הוספה, **לחץ על "שיוך לטרייד"**
2. **בדוק:**
   - ✓ שדה "טיקר" נעלם
   - ✓ שדה "טרייד" מופיע
   - ✓ ההסבר מתחת לשדה חשבון משתנה ל-"חובה כאשר משויך לטרייד"

**תוצאה צפויה:** השדות מתחלפים באופן דינמי.

---

### בדיקה 4: החלפה חזרה (טרייד → טיקר)
**מטרה:** וידוא שהחלפה דו-כיוונית

1. במודל הוספה, **לחץ חזרה על "שיוך לטיקר"**
2. **בדוק:**
   - ✓ שדה "טרייד" נעלם
   - ✓ שדה "טיקר" מופיע שוב
   - ✓ ההסבר משתנה חזרה ל-"אופציונלי כאשר משויך לטיקר"

**תוצאה צפויה:** החלפה חזרה עובדת תקין.

---

### בדיקה 5: שמירת עסקה עם טיקר (יצירה חדשה)
**מטרה:** בדיקה שניתן ליצור execution עם ticker_id בלבד

1. במודל הוספה, בחר **"שיוך לטיקר"**
2. מלא את כל השדות:
   - טיקר: בחר טיקר קיים
   - סוג עסקה: קניה
   - כמות: 50
   - מחיר: 150
   - תאריך: תאריך כלשהו
   - חשבון: **השאר ריק** (אופציונלי)
   - עמלה: 0
3. **לחץ "הוסף עסקה"**

**תוצאה צפויה:**
- ✓ הודעת הצלחה "עסקה נוספה בהצלחה"
- ✓ המודל נסגר
- ✓ הטבלה מתרעננת
- ✓ העסקה החדשה מופיעה בטבלה

**בדיקה נוספת בקונסול:**
```javascript
// Console → Network → בדוק את ה-Payload של POST /api/executions/
// צריך להכיל:
{
  "ticker_id": <מספר>,
  "trade_id": null,
  "trading_account_id": null,
  ...
}
```

---

### בדיקה 6: ולידציה - ניסיון לשמור בלי טיקר
**מטרה:** וידוא שהולידציה עובדת

1. במודל הוספה, בחר **"שיוך לטיקר"**
2. מלא הכל **חוץ מטיקר**
3. לחץ "הוסף עסקה"

**תוצאה צפויה:**
- ✓ הודעת שגיאה: "חובה לבחור טיקר"
- ✓ המודל לא נסגר
- ✓ הטופס לא נשלח

---

### בדיקה 7: שמירת עסקה עם טרייד (מצב רגיל)
**מטרה:** וידוא שמצב trade עובד

1. במודל הוספה, בחר **"שיוך לטרייד"**
2. מלא:
   - טרייד: בחר טרייד קיים
   - חשבון: **חובה** - בחר חשבון
   - שאר השדות
3. לחץ "הוסף עסקה"

**תוצאה צפויה:**
- ✓ הצלחה
- ✓ Payload מכיל: `trade_id: <מספר>, ticker_id: null`

---

### בדיקה 8: פתיחת עריכת עסקה קיימת
**מטרה:** וידוא שעריכה טוענת נכון

1. בטבלה, לחץ **"פעולות" → "ערוך"** על עסקה קיימת (עם trade_id)
2. **בדוק:**
   - ✓ המודל נפתח
   - ✓ הרדיו באטן "שיוך לטרייד" נבחר ✓
   - ✓ שדה טרייד מוצג ומלא
   - ✓ כל השדות האחרים מלאים נכון

**תוצאה צפויה:** מודל עריכה נפתח עם הנתונים הנכונים.

---

### בדיקה 9: שינוי סוג שיוך בעריכה
**מטרה:** וידוא שניתן להחליף מטרייד לטיקר

1. במודל עריכה (של עסקה עם trade), **לחץ "שיוך לטיקר"**
2. **בדוק:**
   - ✓ שדה טרייד נעלם (ומתרוקן)
   - ✓ שדה טיקר מופיע
3. **בחר טיקר** ולחץ "עדכן עסקה"

**תוצאה צפויה:**
- ✓ העסקה עודכנה
- ✓ כעת יש לה ticker_id במקום trade_id

**⚠️ הערה:** זה שינוי לוגי חשוב - אם עסקה הייתה עם trade והפכת אותה לטיקר בלבד, היא "חזרה" למצב זמני.

---

### בדיקה 10: בדיקת CHECK Constraint (שגיאה מכוונת)
**מטרה:** וידוא שהשרת מונע מצב לא חוקי

**דרך הבדיקה:** דרך Console בדפדפן

1. פתח Console ב-DevTools
2. הרץ:
```javascript
fetch('/api/executions/', {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({
    ticker_id: 1,
    trade_id: 1,  // שניהם מלאים - לא חוקי!
    action: 'buy',
    quantity: 10,
    price: 100,
    date: new Date().toISOString()
  })
}).then(r => r.json()).then(console.log)
```

**תוצאה צפויה:**
- ✓ שגיאה 400
- ✓ הודעה: "עסקה לא יכולה להיות משוייכת גם לטיקר וגם לטרייד"

---

### בדיקה 11: API Endpoint - pending-assignment
**מטרה:** וידוא שה-endpoint מחזיר נתונים

1. פתח דפדפן: `http://localhost:8080/api/executions/pending-assignment`

**תוצאה צפויה:**
```json
{
  "status": "success",
  "data": [...],  // רק עסקאות עם ticker_id
  "count": <מספר>,
  "message": "...",
  "version": "1.0"
}
```

**אם אין עסקאות עם ticker_id:**
- `data: []`
- `count: 0`

**אם יש (אחרי בדיקה 5):**
- `data` יכיל את העסקה שיצרת
- `count: 1`

---

### בדיקה 12: בדיקת Console Errors
**מטרה:** וידוא שאין שגיאות JavaScript

1. בכל שלב, **עקוב אחרי Console**
2. בדוק שאין:
   - ❌ אדום (Errors)
   - ⚠️ צהוב (Warnings) חשובות

**תוצאה צפויה:** Console נקי או עם warnings לא קריטיות בלבד.

## 🐛 מה לעשות אם משהו לא עובד

### אם הדף לא נטען
1. בדוק שהשרת רץ
2. בדוק Console לשגיאות JavaScript
3. נקה cache: Cmd+Shift+R

### אם הרדיו באטן לא מוצג
1. נקה cache
2. בדוק ש-executions.html עודכן: `git diff executions.html`

### אם השדות לא מתחלפים
1. בדוק Console לשגיאות
2. וודא ש-`toggleAssignmentFields` קיימת: `typeof window.toggleAssignmentFields` (צריך 'function')

### אם שמירה נכשלת
1. בדוק Network tab → ראה מה השרת החזיר
2. בדוק שהשרת רץ עם הקוד המעודכן
3. אולי צריך restart לשרת
