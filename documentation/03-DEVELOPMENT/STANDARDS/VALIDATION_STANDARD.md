## 🎯 מטרת הסטנדרט
יצירת סטנדרט נוקשה וברור בנושא ולידציה במערכת הפועל בקוד מרכזי אחד המיישם ארכיטקטורה פשוטה ויציבה ככל הניתן.

## 📋 מתי להשתמש בולידציה

### ✅ חובה להשתמש בולידציה:
- כל טופס שמקבל קלט משתמש
- לפני שליחת נתונים לשרת
- בעת עריכת נתונים קיימים
- פעולות הרסניות (מחיקה, ביטול)
- שדות חובה בכל טופס

### ❌ לא נדרש ולידציה:
- עמודים לקריאה בלבד
- עמודים ללא טפסים
- עמודים עם נתונים סטטיים

## 🔧 איך לבחור סוג ולידציה

### **1. Field Validation - `window.validateField()`**
**מתי להשתמש:**
- ולידציה של שדה בודד
- בדיקות פשוטות (required, min, max, pattern)
- ולידציה בזמן אמת

**דוגמה:**
```javascript
window.validateField(input, {
  required: true,
  minLength: 2,
  maxLength: 50,
  pattern: /^[a-zA-Z\s]+$/
});
```

### **2. Form Validation - `window.validateForm()`**
**מתי להשתמש:**
- ולידציה של טופס שלם
- בדיקות בסיסיות של כל השדות
- ולידציה לפני שליחה

**דוגמה:**
```javascript
if (!window.validateForm('addUserForm')) {
  return false;
}
```

### **3. Entity Form - `window.validateEntityForm()`**
**מתי להשתמש:**
- טפסי CRUD עם מספר שדות
- ולידציה מקיפה עם הודעות מפורטות
- טפסים מורכבים

**דוגמה:**
```javascript
const fieldConfigs = [
  {id: 'name', name: 'שם', rules: {required: true, minLength: 2}},
  {id: 'email', name: 'אימייל', rules: {required: true, pattern: /^[^\s@]+@[^\s@]+\.[^\s@]+$/}}
];

const result = window.validateEntityForm('userForm', fieldConfigs);
```

### **4. Cross-field Validation - `window.validateDateRange()`**
**מתי להשתמש:**
- בדיקות בין שדות תאריך
- ולידציה של טווחי ערכים
- בדיקות תלויות בין שדות

**דוגמה:**
```javascript
const isValid = window.validateDateRange('startDate', 'endDate', 'תאריך סיום חייב להיות אחרי תאריך התחלה');
```

### **5. Business Logic - Service Files**
**מתי להשתמש:**
- כללים עסקיים מורכבים
- ולידציה תלוית הקשר
- בדיקות ספציפיות למערכת

**דוגמה:**
```javascript
// alert-service.js
window.alertService.validateAlertStatusCombination(status, isTriggered);
```

### **6. API Validation - Keep in Page**
**מתי להשתמש:**
- בדיקות לפני שליחה לשרת
- ולידציה עם אישור משתמש
- בדיקות תלויות בנתונים מהשרת

**דוגמה:**
```javascript
const result = await window.validateWithConfirmation(
  'אישור מחיקה',
  'האם אתה בטוח שברצונך למחוק?',
  () => window.validateEntityForm('deleteForm', fieldConfigs)
);
```

## 🔗 אינטגרציה עם ולידציה בשרת

### **Client-side Validation = חוויית משתמש**
- מהיר ותגובתי
- מניעת שליחות מיותרות
- הודעות ברורות למשתמש
- חוויית משתמש חלקה

### **Server-side Validation = אבטחה וכללים עסקיים**
- הגנה מפני מניפולציה
- כללים עסקיים מורכבים
- בדיקות תלויות בנתונים
- אבטחת המידע

### **Database Constraints = הגנה אחרונה**
- הגנה ברמת בסיס הנתונים
- מניעת נתונים לא תקינים
- שמירה על שלמות הנתונים

## 🔧 דוגמאות שימוש

### **דוגמה 1: טופס הוספת משתמש**
```javascript
// אתחול ולידציה
window.initializeValidation('addUserForm');

// ולידציה בזמן אמת
document.getElementById('userName').addEventListener('input', function() {
  window.validateField(this, {required: true, minLength: 2});
});

// ולידציה לפני שליחה
function saveUser() {
  const fieldConfigs = [
    {id: 'userName', name: 'שם משתמש', rules: {required: true, minLength: 2}},
    {id: 'userEmail', name: 'אימייל', rules: {required: true, pattern: /^[^\s@]+@[^\s@]+\.[^\s@]+$/}},
    {id: 'userAge', name: 'גיל', rules: {required: false, min: 0, max: 120}}
  ];
  
  const result = window.validateEntityForm('addUserForm', fieldConfigs);
  if (!result.isValid) {
    return false;
  }
  
  // שליחה לשרת...
}
```

### **דוגמה 2: טופס עם תאריכים**
```javascript
// ולידציה בין תאריכים
function validateDateRange() {

    'startDate', 
    'endDate', 
    'תאריך סיום חייב להיות אחרי תאריך התחלה'
  );
  
  if (!isValid) {
    return false;
  }
  
  return true;
}
```
