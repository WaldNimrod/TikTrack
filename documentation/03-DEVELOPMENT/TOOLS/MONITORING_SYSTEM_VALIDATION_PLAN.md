# תוכנית בדיקה מקיפה למערכת הניטור - TikTrack
## Monitoring System Validation Plan

**תאריך יצירה:** 1 נובמבר 2025  
**מטרה:** להבטיח דיוק של 100% במערכת הניטור של איתחול העמודים

---

## 📋 סיכום המערכת

### מה המערכת בודקת:

1. **Missing Scripts** - סקריפטים שצריכים להיות (מתועדים ב-PACKAGE_MANIFEST) אבל לא נטענו
2. **Extra Scripts** - סקריפטים שנטענו אבל לא מתועדים
3. **Duplicate Scripts** - סקריפטים שנטענו מספר פעמים (כפילויות)
4. **Loading Order Issues** - בעיות בסדר טעינה (סקריפט נטען לפני תלויותיו)
5. **Missing Globals** - משתנים גלובליים שצריכים להיות אבל לא קיימים
6. **Version Mismatches** - אי-התאמות גרסאות בין PACKAGE_MANIFEST לבין מה שנטען
7. **Circular Dependencies** - תלויות מעגליות בין חבילות

---

## 🎯 מטרות הבדיקה

### דיוק (Accuracy)
- **False Positives (FP):** מזהה בעיות שלא קיימות - צריך להיות 0%
- **False Negatives (FN):** לא מזהה בעיות שקיימות - צריך להיות 0%
- **True Positives (TP):** מזהה נכון בעיות שקיימות - צריך להיות 100%
- **True Negatives (TN):** מזהה נכון שאין בעיות - צריך להיות 100%

### כיסוי (Coverage)
- בדיקת כל סוגי הבעיות
- בדיקת כל העמודים המתועדים
- בדיקת edge cases
- בדיקת תרחישים ריאליסטיים

---

## 📊 תוכנית בדיקה - 4 שלבים

### שלב 1: בדיקות בסיסיות (Base Tests)
**מטרה:** לוודא שהמערכת עובדת במצבים בסיסיים

#### 1.1 בדיקת עמוד תקין ללא בעיות
**מטרה:** לוודא שלא מזהה בעיות שלא קיימות (False Positives)

**תרחיש:**
- עמוד עם קונפיגורציה מלאה ונכונה
- כל הסקריפטים הנדרשים נטענו
- סדר טעינה נכון
- כל הגלובליים קיימים

**צפוי:**
```javascript
{
  mismatches: 0,
  duplicates: [],
  loadOrderIssues: [],
  mismatchDetails: []
}
```

**אימות:**
- [ ] 0 בעיות מזוהות
- [ ] הודעה "✅ מערכת תקינה - לא נמצאו בעיות!"
- [ ] מודל תוצאות לא מוצג (או מוצג עם 0 בעיות)

---

#### 1.2 בדיקת עמוד עם missing script אחד
**מטרה:** לוודא שמזהה נכון סקריפט חסר

**תרחיש:**
- עמוד עם קונפיגורציה מלאה
- הסרת סקריפט אחד מתוך ה-HTML לפני הבדיקה
- שאר הסקריפטים תקינים

**צפוי:**
```javascript
{
  mismatches: 1,
  mismatchDetails: [{
    type: 'missing_script',
    message: 'סקריפט חסר: X.js - מתועד אבל לא נטען',
    severity: 'error'
  }]
}
```

**אימות:**
- [ ] מזהה בדיוק את הסקריפט החסר
- [ ] הודעת השגיאה נכונה ומדויקת
- [ ] severity: 'error'

---

#### 1.3 בדיקת עמוד עם extra script אחד
**מטרה:** לוודא שמזהה נכון סקריפט נוסף לא מתועד

**תרחיש:**
- עמוד תקין
- הוספת סקריפט שלא מתועד ב-PACKAGE_MANIFEST

**צפוי:**
```javascript
{
  mismatches: 1,
  mismatchDetails: [{
    type: 'extra_script',
    message: 'סקריפט נוסף: Y.js - נטען אבל לא מתועד',
    severity: 'warning'
  }]
}
```

**אימות:**
- [ ] מזהה את הסקריפט הנוסף
- [ ] severity: 'warning' (לא error)
- [ ] לא מזהה false positives אחרים

---

#### 1.4 בדיקת כפילות סקריפט
**מטרה:** לוודא שמזהה נכון כפילויות

**תרחיש:**
- עמוד עם סקריפט שנטען פעמיים ב-HTML

**צפוי:**
```javascript
{
  duplicates: ['Z.js'],
  mismatches: 1,
  mismatchDetails: [{
    type: 'duplicate_script',
    message: 'כפילות: Z.js נטען 2 פעמים',
    severity: 'error'
  }]
}
```

**אימות:**
- [ ] מזהה את הכפילות
- [ ] מספר הכפילויות מדויק
- [ ] נכנס גם ל-duplicates וגם ל-mismatchDetails

---

### שלב 2: בדיקות מתקדמות (Advanced Tests)
**מטרה:** לוודא שהמערכת מזהה בעיות מורכבות

#### 2.1 בדיקת סדר טעינה שגוי
**תרחיש:**
- עמוד עם סקריפט שנטען לפני תלות שלו

**צפוי:**
```javascript
{
  loadOrderIssues: [{
    type: 'loading_order',
    message: 'סדר טעינה שגוי: X.js נטען במקום N אבל נמצא במקום M',
    severity: 'error'
  }]
}
```

**אימות:**
- [ ] מזהה את הבעיה
- [ ] ההודעה מדויקת ומכילה מידע רלוונטי

---

#### 2.2 בדיקת missing global
**תרחיש:**
- עמוד שמצפה ל-global variable שלא קיים

**צפוי:**
```javascript
{
  mismatches: 1,
  mismatchDetails: [{
    type: 'missing_global',
    message: 'גלובל חסר: window.X - נדרש לפי התיעוד אבל לא זמין',
    severity: 'error'
  }]
}
```

**אימות:**
- [ ] מזהה את הגלובל החסר
- [ ] ההודעה מדויקת

---

#### 2.3 בדיקת multiple issues
**תרחיש:**
- עמוד עם כמה בעיות שונות בו-זמנית

**צפוי:**
- מספר בעיות מדויק
- כל בעיה מזוהה נכון
- סיכום נכון

**אימות:**
- [ ] ספירה נכונה של כל הבעיות
- [ ] כל בעיה מופיעה ב-mismatchDetails
- [ ] סכום הבעיות תואם לספירה

---

### שלב 3: בדיקות Edge Cases
**מטרה:** לוודא שהמערכת מטפלת נכון במקרים מיוחדים

#### 3.1 עמוד ללא קונפיגורציה
**תרחיש:**
- קריאה ל-`runDetailedPageScan` לעמוד שלא קיים ב-PAGE_CONFIGS

**צפוי:**
- שגיאה או תוצאה ריקה
- הודעה ברורה למשתמש

**אימות:**
- [ ] לא קורס
- [ ] מחזיר תוצאה תקינה

---

#### 3.2 עמוד עם scripts עם query strings
**תרחיש:**
- סקריפטים עם `?v=hash` או פרמטרים אחרים

**צפוי:**
- מזהה נכון את הסקריפטים למרות הפרמטרים
- לא מזהה כפילויות שגויות

**אימות:**
- [ ] מזהה נכון
- [ ] לא מזהה false positives

---

#### 3.3 עמוד עם סקריפטים דינמיים
**תרחיש:**
- סקריפטים שנוספים דינמית אחרי טעינת הדף

**צפוי:**
- אולי לא יזהה (תלוי מתי הקוד רץ)
- או שיזהה אחרי `waitForPageFullyLoaded`

**אימות:**
- [ ] מתנהג עקבי
- [ ] ההודעות ברורות

---

#### 3.4 עמוד עם package dependencies מעגליות
**תרחיש:**
- PACKAGE_MANIFEST עם circular dependencies

**צפוי:**
```javascript
// צריך לזהות circular dependencies
```

**אימות:**
- [ ] מזהה את התלויות המעגליות
- [ ] לא קורס

---

#### 3.5 עמוד עם version mismatch
**תרחיש:**
- סקריפט עם version שונה מה-PACKAGE_MANIFEST

**צפוי:**
- מזהה version mismatch
- warning (לא error)

**אימות:**
- [ ] מזהה את ההבדל
- [ ] severity נכון

### שלב 4: בדיקות אינטגרציה וטעינת דף
**מטרה:** לוודא שהמערכת עובדת נכון במצבים ריאליסטיים

**אימות:**
- [ ] ממתין נכון

**אימות:**
- [ ] מזהה את הבעיה
