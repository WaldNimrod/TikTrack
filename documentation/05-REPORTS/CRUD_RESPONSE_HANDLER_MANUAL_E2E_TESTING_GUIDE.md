# מדריך בדיקות E2E ידניות מלאות - CRUD Response Handler

**תאריך יצירה:** 25 בנובמבר 2025  
**גרסה:** 1.0.0  
**סטטוס:** 🔄 בתהליך

---

## מטרת המדריך

מדריך זה מספק הוראות מפורטות לביצוע בדיקות E2E ידניות מלאות למערכת CRUD Response Handler בכל 7 העמודים המרכזיים.

---

## כללי בדיקה

### לפני כל בדיקה:
1. פתח את הקונסולה (F12) ובדוק ש-`window.CRUDResponseHandler` זמין
2. בדוק שאין שגיאות בקונסולה
3. ודא שהעמוד נטען במלואו

### מה לבדוק בכל פעולה:
1. ✅ **CRUDResponseHandler מטפל בתגובה** - בדוק בקונסולה שהפונקציה נקראה
2. ✅ **הודעות מוצגות** - הודעת הצלחה/שגיאה מוצגת
3. ✅ **מודלים נסגרים** - מודל נסגר אחרי פעולה מוצלחת (לא נסגר על שגיאת ולידציה)
4. ✅ **טבלאות מתעדכנות** - הטבלה מתעדכנת עם הנתונים החדשים
5. ✅ **שגיאות מטופלות** - שגיאות ולידציה (400) ושגיאות מערכת (500) מטופלות נכון

## 1. Trades (טריידים) - `trades.html`

### בדיקת יצירה (CREATE):

1. **פתח את העמוד** `trades.html`
2. **לחץ על "הוסף טרייד"**
3. **מלא את הטופס:**
   - טיקר: בחר טיקר קיים (למשל: TSLA)
   - חשבון מסחר: בחר חשבון קיים
   - צד: Long או Short
   - סוג השקעה: בחר סוג השקעה
   - סטטוס: פתוח
   - מחיר כניסה: 250
   - כמות: 10
4. **לחץ על "שמור"**
5. **בדוק:**
   - [ ] הודעת הצלחה מוצגת
   - [ ] המודל נסגר
   - [ ] הטבלה מתעדכנת עם הטרייד החדש
   - [ ] הסטטיסטיקות מתעדכנות

### בדיקת עדכון (UPDATE):

1. **מצא טרייד קיים** בטבלה
2. **לחץ על כפתור "פעולות"** → "ערוך"
3. **ערוך לפחות 2 שדות:**
   - שדה 1: מחיר כניסה (למשל: 255)
   - שדה 2: כמות (למשל: 15)
4. **לחץ על "שמור"**
5. **בדוק:**
   - [ ] הודעת הצלחה מוצגת
   - [ ] המודל נסגר
   - [ ] הטבלה מתעדכנת עם הערכים החדשים
   - [ ] השינויים נראים בטבלה

### בדיקת מחיקה (DELETE):

1. **מצא טרייד קיים** בטבלה
2. **לחץ על כפתור "פעולות"** → "מחק"
3. **אשר את המחיקה** (אם יש דיאלוג אישור)
4. **בדוק:**
   - [ ] הודעת הצלחה מוצגת
   - [ ] הטרייד נמחק מהטבלה
   - [ ] הסטטיסטיקות מתעדכנות

### בדיקת שגיאות ולידציה (400):

1. **פתח מודל יצירה**
2. **מלא טופס עם נתונים לא תקינים** (למשל: מחיר שלילי, כמות 0)
3. **לחץ על "שמור"**
4. **בדוק:**
   - [ ] הודעת שגיאה מוצגת עם פרטי השגיאה
   - [ ] המודל **לא** נסגר (נכון!)
   - [ ] הטבלה **לא** מתעדכנת

## הערות חשובות

1. **בעיית ולידציה בשרת** - ב-`trades.html` יש בעיה בנורמליזציה של ערכים (`investment_type` ו-`side`). זה לא בעיה של CRUDResponseHandler, אלא בעיה של נורמליזציה לפני שליחה.

2. **טריידים קיימים** - כדי לבדוק עדכון ומחיקה, צריך שיהיו טריידים קיימים במערכת. אם אין, צריך ליצור קודם.

3. **בדיקת קונסולה** - בכל פעולה, בדוק בקונסולה ש-`CRUDResponseHandler` נקרא והתגובה מטופלת נכון.
