# רשימת בדיקות CRUD - TikTrack

**תאריך יצירה:** 1 בדצמבר 2025  
**גרסה:** 1.0.0  
**סטטוס:** 🔄 בתהליך

---

## 📋 הוראות שימוש

רשימה זו מיועדת לבדיקה ידנית של כל פעולות CRUD ב-8 העמודים המרכזיים.

**דרישות:**
- משתמש מנהל (admin) - פרטי התחברות מופיעים בעמוד הכניסה
- שרת פועל על `http://localhost:8080`
- דפדפן עם קונסול פתוח לזיהוי שגיאות

**שיטת עבודה:**
1. עבור עמוד אחר עמוד לפי הסדר
2. סמן ✅ עבור כל בדיקה שעברה בהצלחה
3. סמן ❌ עבור כל בדיקה שנכשלה ותעד את הבעיה
4. תיעד כל בעיה בקובץ `CRUD_REVIVAL_STATUS.md`

---

## 1. trades.html - ניהול טריידים

### בדיקות בסיסיות
- [ ] **טעינת עמוד**: פתיחת `http://localhost:8080/trades.html`
- [ ] **אין שגיאות בקונסול**: בדיקת קונסול הדפדפן - אין שגיאות JavaScript
- [ ] **טבלה נטענת**: הטבלה מציגה רשומות (אם יש נתונים)

### בדיקת Read (קריאה)
- [ ] **טעינת נתונים**: הטבלה מציגה רשומות טריידים
- [ ] **עמודות נכונות**: כל העמודות מוצגות (טיקר, מחיר, שינוי, פוזיציה, P/L%, P/L, סטטוס, סוג, צד, חשבון, תאריכים)
- [ ] **נתונים נכונים**: הנתונים בטבלה תואמים למציאות

### בדיקת Create (יצירה)
- [ ] **כפתור "הוסף" קיים**: כפתור "הוסף טרייד" מופיע בכותרת הטבלה
- [ ] **לחיצה על "הוסף"**: לחיצה על הכפתור פותחת מודל `tradesModal` במצב `add`
- [ ] **שדות בטופס**: כל השדות הנדרשים מופיעים (טיקר, חשבון, צד, סוג, כמות, מחיר, וכו')
- [ ] **מילוי טופס**: מילוי כל השדות הנדרשים
- [ ] **שמירה**: לחיצה על "שמור" שומרת את הרשומה
- [ ] **הודעת הצלחה**: מופיעה הודעת הצלחה "טרייד נוסף בהצלחה"
- [ ] **מודל נסגר**: המודל נסגר אוטומטית
- [ ] **רשומה בטבלה**: הרשומה החדשה מופיעה בטבלה
- [ ] **נתונים נכונים**: הנתונים שנשמרו תואמים למה שהוזן בטופס

### בדיקת Update (עדכון)
- [ ] **כפתור "ערוך" קיים**: כפתור "ערוך" מופיע בכל שורה בטבלה
- [ ] **לחיצה על "ערוך"**: לחיצה על הכפתור פותחת מודל `tradesModal` במצב `edit`
- [ ] **נתונים בטופס**: כל השדות מלאים בנתונים הקיימים של הרשומה
- [ ] **שינוי נתונים**: שינוי של שדה אחד לפחות (למשל: כמות או מחיר)
- [ ] **שמירה**: לחיצה על "שמור" שומרת את השינויים
- [ ] **הודעת הצלחה**: מופיעה הודעת הצלחה "טרייד עודכן בהצלחה"
- [ ] **מודל נסגר**: המודל נסגר אוטומטית
- [ ] **שינויים בטבלה**: השינויים מופיעים בטבלה
- [ ] **נתונים נכונים**: הנתונים המעודכנים תואמים למה שהוזן בטופס

### בדיקת Delete (מחיקה)
- [ ] **כפתור "מחק" קיים**: כפתור "מחק" מופיע בכל שורה בטבלה
- [ ] **לחיצה על "מחק"**: לחיצה על הכפתור מציגה בדיקת פריטים מקושרים
- [ ] **אישור מחיקה**: אם אין פריטים מקושרים, מופיע חלון אישור מחיקה
- [ ] **ביטול מחיקה**: ביטול המחיקה לא מוחק את הרשומה
- [ ] **אישור מחיקה**: אישור המחיקה מוחק את הרשומה
- [ ] **הודעת הצלחה**: מופיעה הודעת הצלחה "טרייד נמחק בהצלחה"
- [ ] **רשומה נמחקה**: הרשומה נעלמה מהטבלה
- [ ] **רענון טבלה**: הטבלה מתעדכנת אוטומטית

### בדיקת מערכות משולבות
- [ ] **ModalManagerV2**: המודל נפתח דרך ModalManagerV2
- [ ] **CRUDResponseHandler**: הודעות הצלחה/שגיאה מופיעות דרך CRUDResponseHandler
- [ ] **DataCollectionService**: נתונים נאספים דרך DataCollectionService
- [ ] **checkLinkedItemsBeforeAction**: בדיקת פריטים מקושרים לפני מחיקה
