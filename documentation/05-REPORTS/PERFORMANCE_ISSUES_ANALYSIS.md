## 🔍 סיבות שורשיות

### 1. אין async/defer על סקריפטים
**מיקום:** כל קבצי HTML (index.html, trades.html, וכו')

**בעיה:**
```html
<!-- ❌ נוכחי - blocking -->
<script src="scripts/api-config.js?v=1.0.0"></script>
<script src="scripts/global-favicon.js?v=1.0.0"></script>
<!-- ... 116+ סקריפטים נוספים -->
```

**פתרון:**
```html
<!-- ✅ מומלץ - async/defer -->
<script src="scripts/api-config.js?v=1.0.0" defer></script>
<script src="scripts/global-favicon.js?v=1.0.0" defer></script>
<!-- סקריפטים קריטיים: defer -->
<!-- סקריפטים לא קריטיים: async -->
```

---

### 2. אין lazy loading
**בעיה:**
- כל הסקריפטים נטענים מיד גם אם לא נדרשים
- אין conditional loading לפי עמוד
- אין dynamic imports

**פתרון:**
- טעינה דינמית של מודולים לפי צורך
- שימוש ב-`loadScriptOnce()` עם lazy loading
- טעינת מודולים רק כשנדרשים

---

### 3. אין code splitting
**בעיה:**
- כל הקוד נטען יחד
- אין הפרדה בין קוד קריטי לקוד לא קריטי
- אין chunking

**פתרון:**
- הפרדת קוד קריטי (base) מקוד לא קריטי
- טעינת קוד לא קריטי רק כשנדרש
- שימוש ב-dynamic imports

---

### 4. אין bundling
**בעיה:**
- כל מודול נטען כקובץ נפרד
- 116-120 בקשות HTTP נפרדות
- אין איחוד קבצים

**פתרון:**
- איחוד קבצים קטנים לקבצים גדולים יותר
- הפחתת מספר הבקשות
- שימוש ב-bundler (webpack, rollup, וכו')

## 💡 המלצות לתיקון

### עדיפות גבוהה (קריטי):

1. **הוספת async/defer לסקריפטים**
   - סקריפטים קריטיים: `defer`
   - סקריפטים לא קריטיים: `async`
   - סקריפטים תלויים: `defer` בסדר הנכון

2. **הפחתת מספר הסקריפטים**
   - איחוד קבצים קטנים
   - שימוש ב-bundling
   - טעינה דינמית של מודולים

3. **Lazy loading**
   - טעינת מודולים רק כשנדרשים
   - שימוש ב-`loadScriptOnce()` עם lazy loading
   - conditional loading לפי עמוד

### עדיפות בינונית:

4. **Code splitting**
   - הפרדת קוד קריטי מקוד לא קריטי
   - טעינת קוד לא קריטי רק כשנדרש
   - שימוש ב-dynamic imports

5. **אופטימיזציה של משאבים**
   - Minification מתקדם
   - Tree shaking
   - Compression (gzip/brotli)

### עדיפות נמוכה:

6. **CDN לקבצים סטטיים**
7. **Service Worker לקאשינג**
8. **HTTP/2 Server Push**
