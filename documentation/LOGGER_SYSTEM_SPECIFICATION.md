# אפיון מערכת הלוגר - TikTrack

## סקירה כללית

מערכת הלוגר של TikTrack מספקת לוגים מקיפים עם שליטה מלאה על רמות הלוג, שמירה מקומית ושליחה לשרת. המערכת תוכננה להיות יעילה במצב ייצור ומפורטת במצב פיתוח.

## מצבי פעולה

### מצב DEBUG
**מזהה**: `Logger.DEBUG_MODE = true`
**תנאים**:
- `window.location.hostname === 'localhost'`
- `window.location.hostname === '127.0.0.1'`
- `window.location.search.includes('debug=true')`
- `window.location.search.includes('dev=true')`

**התנהגות**:
- ✅ כל הלוגים מוצגים בקונסולה
- ✅ כל הלוגים נשמרים ב-localStorage
- ✅ כל הלוגים נשלחים לשרת
- ✅ ניטור ביצועים פעיל

### מצב ייצור (Production)
**מזהה**: `Logger.DEBUG_MODE = false`
**התנהגות**:
- ❌ רק שגיאות (ERROR + CRITICAL) מוצגות בקונסולה
- ❌ לא שומרים ב-localStorage
- ❌ רק שגיאות נשלחות לשרת
- ❌ ניטור ביצועים לא פעיל

## רמות לוג

```javascript
Logger.LogLevel = {
    DEBUG: 0,      // מידע מפורט לפיתוח
    INFO: 1,       // מידע כללי
    WARN: 2,       // אזהרות
    ERROR: 3,      // שגיאות
    CRITICAL: 4    // שגיאות קריטיות
};
```

## מיקומי לוגים

### 1. קונסולה של הדפדפן
**מיקום**: Developer Tools > Console
**נגישות**: ישירה
**תוכן**: 
- מצב DEBUG: כל הלוגים
- מצב ייצור: רק שגיאות

### 2. localStorage
**מיקום**: Developer Tools > Application > Local Storage > `tiktrack_pending_logs`
**נגישות**: ישירה
**תוכן**: 
- מצב DEBUG: עד 100 לוגים אחרונים
- מצב ייצור: לא נשמר

### 3. שרת לוגים
**מיקום**: `Backend/server_detailed.log`
**נגישות**: קובץ לוגים של השרת
**תוכן**: 
- מצב DEBUG: כל הלוגים
- מצב ייצור: רק שגיאות

### 4. בסיס נתונים
**מיקום**: טבלת `logs` (לא קיימת כרגע)
**נגישות**: לא זמין
**תוכן**: לא נשמר

## איך לגשת ללוגים

### למפתחים
1. **פתח את הדפדפן במצב DEBUG**
2. **פתח Developer Tools (F12)**
3. **לך ל-Console** - תראה את כל הלוגים
4. **לך ל-Application > Local Storage** - תראה לוגים שמורים

### לניתוח מתקדם
1. **בדוק את קובץ הלוגים של השרת**:
   ```bash
   tail -f Backend/server_detailed.log
   ```

2. **בדוק localStorage**:
   ```javascript
   // בקונסולה של הדפדפן
   JSON.parse(localStorage.getItem('tiktrack_pending_logs'))
   ```

## פתרון בעיות נפוצות

### העמודים לא נטענים
**סיבה**: יותר מדי לוגים יוצרים עומס
**פתרון**: 
1. עבור למצב ייצור (הסר `?debug=true`)
2. או הגבל לוגים במערכת ההעדפות

### אין לוגים בקונסולה
**סיבה**: במצב ייצור
**פתרון**: הוסף `?debug=true` ל-URL

### לוגים לא נשמרים
**סיבה**: במצב ייצור localStorage לא פעיל
**פתרון**: עבור למצב DEBUG

### שגיאות 429 (Too Many Requests)
**סיבה**: יותר מדי לוגים נשלחים לשרת
**פתרון**: 
1. עבור למצב ייצור
2. או הגבל לוגים במערכת ההעדפות

## המלצות למפתחים

### במהלך פיתוח
1. **השתמש במצב DEBUG** עם `?debug=true`
2. **בדוק את הקונסולה** תמיד
3. **השתמש ב-localStorage** לניתוח מתקדם

### לפני פריסה
1. **בדוק במצב ייצור** (ללא `?debug=true`)
2. **ודא שאין לוגים מיותרים**
3. **בדוק ביצועים** עם ניקוי מטמון

### בעת דיבג
1. **עבור למצב DEBUG**
2. **נקה את המטמון** (Ctrl+Shift+R)
3. **בדוק את הקונסולה**
4. **בדוק את localStorage**
