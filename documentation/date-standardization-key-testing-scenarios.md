## 🎯 תרחישי בדיקה מרכזיים

### 1. דף Trades - תצוגה ומיון תאריכים ⭐⭐⭐

**מה לבדוק:**

1. **תצוגת תאריכים:**
   - פתח `/trades`
   - ודא שעמודת "נוצר ב:" מציגה תאריכים בפורמט עברי (dd.mm.yyyy HH:MM)
   - ודא שעמודת "עודכן" מציגה תאריכים (אם קיים) או "לא זמין"
   - ודא שתאריכים null מוצגים כ-"לא זמין" או "-"

2. **מיון לפי תאריך:**
   - לחץ על כותרת "נוצר ב:" - המיון אמור לעבוד (מהחדש לישן ולהיפך)
   - לחץ על כותרת "עודכן" - המיון אמור לעבוד
   - ודא שהמיון עובד נכון גם עם תאריכים null

3. **הוספה ועריכה:**
   - הוסף trade חדש - ודא שתאריך יצירה מוצג נכון בטבלה
   - ערוך trade קיים - ודא שתאריך עדכון מעודכן נכון

### 8. בדיקת API Endpoints - נרמול תאריכים ⭐⭐

**מה לבדוק:**

1. **בדיקת Response:**
   - פתח Developer Tools → Network
   - בצע פעולות שונות (טעינת דפים, הוספה, עריכה)
   - בדוק שה-responses מכילים `DateEnvelope` תקין:

     ```json
     {
       "created_at": {
         "utc": "2025-11-21T10:00:00Z",
         "epochMs": 1732186800000,
         "local": "2025-11-21T12:00:00+02:00",
         "timezone": "Asia/Jerusalem",
         "display": "21.11.2025 12:00"
       }
     }
     ```

2. **בדיקת Endpoints שתוקנו:**
   - `GET /api/tickers` - ודא ש-`created_at` ו-`updated_at` הם `DateEnvelope`
   - `POST /api/plan-conditions/test` - ודא ש-response מכיל `DateEnvelope`
   - `PUT /api/trade-conditions/<id>` - ודא ש-response מכיל `DateEnvelope`

## 📝 הערות חשובות

1. **פורמט תאריך:** כל התאריכים אמורים להיות בפורמט עברי (dd.mm.yyyy HH:MM)
2. **Timezone:** כל התאריכים אמורים להיות ב-timezone של המשתמש (Asia/Jerusalem)
3. **תאריכים null:** תאריכים null אמורים להיות מוצגים כ-"לא זמין" או "-"
4. **DateEnvelope:** כל התאריכים מהשרת אמורים להיות בפורמט `DateEnvelope`

---

## 🐛 בעיות נפוצות לבדיקה

1. **תאריך מציג N/A במקום תאריך:**
   - בדוק שהשרת מחזיר `DateEnvelope` תקין
   - בדוק שהקוד ב-frontend משתמש ב-`FieldRendererService.renderDate`

2. **מיון לא עובד:**
   - בדוק שהטבלה רשומה ב-`UnifiedTableSystem`
   - בדוק שהעמודה מוגדרת כ-`dateEnvelope` ב-`table-mappings.js`

3. **תאריך מוצג ב-timezone שגוי:**
   - בדוק שה-`DateEnvelope` מכיל `timezone` נכון
   - בדוק שהקוד משתמש ב-`dateUtils` להמרת timezone
