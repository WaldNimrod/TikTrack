## 🔗 מערכת Dependencies (חדש!)

### **📋 Dependencies Matrix:**
מציג את מטריצת התלויות של המערכת החדשة:

```
tickers    → dashboard
trades     → tickers, dashboard  
executions → trades, dashboard
preferences → (עצמאי)
currencies → (עצמאי)
```

### **⚡ TTL Strategy:**
מציג את אסטרטגיית הזמן לכל סוג נתונים:

- **🔥 קריטי (30 שניות)**: `tickers`, `trades`, `executions`
- **⚡ רגיל (5 דקות)**: `preferences`
- **🏗️ סטטי (שעה)**: `currencies`

### **🧪 Invalidation Testing:**
כלי לבדיקת ביטול cache לפי dependency:

1. **בחר dependency** מהרשימה הנפתחת
2. **לחץ "Invalidate Dependency"**
3. **רואה תוצאות** בזמן אמת

## 🛠️ פעולות Cache

### **🧹 ניקוי Cache**
- **מטרה**: מחיקת כל רשומות הcache  
- **שימוש**: למצבי פיתוח או תיקון בעיות
- **זמן**: מיידי
- **אחרי**: כל הבקשות הבאות ילכו לבסיס הנתונים

### **🏥 בדיקת בריאות**
- **מטרה**: אימות שמערכת הcache עובדת תקין
- **בודק**: זיכרון, hit rate, expired entries
- **תוצאה**: בריא/אזהרה/שגיאה

### **📊 סטטיסטיקות**
- **מטרה**: נתונים מפורטים על הביצועים
- **כולל**: רשומות, hits, misses, invalidations
- **עדכון**: אוטומטי כל 5 שניות

### **🔄 רענון נתונים**
- **מטרה**: עדכון כל הנתונים מהשרת  
- **שימוש**: לוודא שרואים מידע עדכני
- **זמן**: 2-3 שניות

## 🧪 בדיקות Cache

### **💾 שמירה ב-Cache (סימולציה)**
- **מה זה עושה**: מדמה פעולת cache set
- **שדות**: מפתח, ערך, TTL
- **הערה**: זוהי סימולציה - השרת האמיתי משתמש בdependencies

### **🔍 קריאה מ-Cache (סימולציה)**  
- **מה זה עושה**: מדמה פעולת cache get
- **שדות**: מפתח
- **תוצאה**: הודעת cache miss (נורמל במערכת החדשה)

### **🗑️ מחיקה מ-Cache (סימולציה)**
- **מה זה עושה**: מדמה פעולת cache delete  
- **שדות**: מפתח
- **הערה**: במערכת האמיתית משתמשים ב-dependency invalidation

## 🚨 התראות ואבחונים

### **🔍 אבחון בעיות נפוצות:**

#### **Hit Rate נמוך (<50%)**
- **סיבות אפשריות**: cache TTL קצר מדי, invalidations תכופים מדי
- **פתרון**: בדוק TTL strategy, וודא שdependencies נכונים

#### **שימוש זיכרון גבוה (>80%)**
- **סיבות אפשריות**: יותר מדי cache entries, lack של cleanup
- **פתרון**: נקה cache, בדוק memory optimization settings

#### **Invalidation לא עובד**
- **סיבות אפשריות**: dependency שגוי, בעיה בAPI
- **פתרון**: בדוק שהdependency נכון ברשימה, צפה בלוגים

#### **שגיאות API**
- **סיבות אפשריות**: השרת לא רץ, בעיה ב-endpoints  
- **פתרון**: וודא שהשרת רץ על port 8080

### **📋 רשימת endpoints שהמערכת משתמשת:**
- **`/api/cache/stats`** - סטטיסטיקות cache
- **`/api/cache/health`** - בריאות המערכת  
- **`/api/cache/status`** - מצב כללי
- **`/api/cache/clear`** - ניקוי cache
- **`/api/cache/invalidate`** - ביטול לפי dependency
