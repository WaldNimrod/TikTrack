# ARCHITECT DECISION TEMPLATE STANDARD

## Decision
All new or updated files under `_COMMUNICATION/_Architects_Decisions/` must use one canonical structure:

- Template file: `_COMMUNICATION/_Architects_Decisions/ARCHITECT_DECISION_TEMPLATE.md`
- Procedure file: `documentation/05-PROCEDURES/TT2_ARCHITECT_DECISION_TEMPLATE_PROCEDURE.md`

## Scope

In scope:
- New architect decisions
- Existing architect decisions that are being edited now

Out of scope:
- Historical decisions that are not being modified

## Binding Rules
1. No new decision file is valid without the canonical template structure.
2. Retroactive rewrite of all historical decisions is prohibited.
3. Historical decisions are normalized only when reopened for update.
4. Team 70 executes documentation updates.
5. Team 10 is the gateway approval gate.
6. Team 90 validates format and authority integrity.

## Rationale
One template prevents authority drift, reduces format variance, and keeps decisions traceable across governance audits.
