## 3. מבנה העמוד

### 3.1 כותרת

- **כותרת עמוד:** "דשבורד נתונים" (או "נתוני שוק" — לפי החלטת UX).
- שימוש ב־header/תבנית עמוד סטנדרטית (כמו בשאר העמודים הפיניקס).

### 3.2 טבלה 1 — רשימת שערי חליפין (Exchange Rates)

**תיאור:** טבלה שמציגה את **כל השערים** שהמערכת מחזיקה כרגע.

**עמודות (מינימום):**

| עמודה | תיאור | מקור נתונים |
|--------|--------|--------------|
| זוג מטבעות | from_currency / to_currency (למשל USD/ILS, EUR/ILS) | `market_data.exchange_rates` |
| שער עדכני | conversion_rate (ערך נוכחי) | אותו מקור |
| זמן עדכון אחרון | last_sync_time (תאריך+שעה) | אותו מקור |

**מקור API:** `GET /api/v1/reference/exchange-rates` — כבר קיים. השדות ב-response: רשימת פריטים עם `from_currency`, `to_currency`, `conversion_rate`, `last_sync_time` (או מקביל לפי הסכמה הנוכחית). יש להתאים את שמות העמודות לשמות השדות ב-API.

**התנהגות:** טעינה בעת כניסה לעמוד; רענון (אופציונלי) בלחיצת כפתור או אוטומטי לפי staleness.

### 3.3 טבלה 2 — היסטוריה של שער נבחר

**תיאור:** טבלה שמציגה **היסטוריה** של שער חליפין אחד — לפי בחירת המשתמש.

**בכותרת הטבלה (או מעליה):**
- **דרופדאון (Dropdown):** לבחירת השער (זוג מטבעות).  
  אפשרויות: רשימת הזוגות הזמינים במערכת (כמו בטבלה 1), למשל USD/ILS, USD/EUR, EUR/ILS וכו'.

**עמודות (מינימום):**

| עמודה | תיאור |
|--------|--------|
| תאריך/שעה | as_of (או timestamp של הדגימה) |
| שער | conversion_rate באותה נקודת זמן |

**מקור API:**  
כרגע ב-DB יש רק **ערך עדכני** לכל זוג (טבלה `market_data.exchange_rates` — ON CONFLICT DO UPDATE).  
לכן אחת מהאפשרויות:
- **אופציה א:** Backend מוסיף טבלה/לוג היסטוריה (למשל `market_data.exchange_rates_history`) וסקריפט ה-sync כותב גם שורות היסטוריה; ו-API חדש `GET /api/v1/reference/exchange-rates/{from}/{to}/history` מחזיר רשימת נקודות.  
- **אופציה ב:** בלי היסטוריה ב-DB — טבלה 2 מוצגת כ"לא זמין" או מציגה רק את השורה הנוכחית (עד שייושם backend להיסטוריה).

**במפרט זה:** מגדירים את הטבלה + הדרופדאון. יישום הטבלה עם נתוני היסטוריה אמיתיים מותנה ביכולת Backend (Team 20 / ארכיטקטורה).
