## ממצאים (לפי עדיפות)

### חוסמים (לעדכן מיד)

**1. עמודת "אלמנט מקושר" — כל העמודים הרלוונטיים**  
- **נוכחי:** מוצג איקון + סוג בלבד.  
- **נדרש:** להציג את **הרשומה הספציפית** (שם/סמל, למשל AAPL) **+ קישור** למודול הפרטים של הרשומה המקושרת.  
- **טכני:** וודא ש־`linked_entity_display` / `target_display_name` מגיעים מ־API ומוצגים; הוסף לינק (למשל `data-entity` + id) שפותח את מודול הפרטים של הישות המקושרת.

**2. קובץ מצורף — D35 הערות**  
- **נוכחי:** קובץ מצורף לא מוצג לא בטבלה ולא במודול פרטים.  
- **נדרש:** הצגת קבצים מצורפים **בטבלה** (עמודה/אינדיקציה) ו**במודול פרטי הערה** (רשימה + פתח/הורד).  
- **טכני:** `notesTableInit` — עמודה/תא לקבצים; בפרטי הערה — `buildAttachmentsHtml` + טעינת קבצים מהשרת והצגה.

**3. רענון טבלה אחרי עדכון (חמור)**  
- **נוכחי:** אחרי ביצוע עדכון לרשומה — הטבלה **לא** מתעדכנת במידע החדש.  
- **נדרש:** אחרי שמירת עריכה — **רענון הטבלה** (או עדכון השורה) כך שהנתונים החדשים יופיעו **מיד** בלי ריענון דף.  
- **טכני:** לאחר `PUT`/`PATCH` מוצלח — לקרוא ל־`loadTickersData`/`refreshAlertsTable`/`refreshNotesTable` (או מקביל) ולעדכן את ה־DOM.

**6. הוספת הערה — ממשק קישור**  
- **נוכחי:** מציג "אופציונלי" ולא מאפשר קישור לישות **ספציפית**.  
- **נדרש:** קישור **חובה** לישות ספציפית; ממשק לבחירת ישות מקושרת (דרופדאון/חיפוש לפי טיקר/חשבון/תוכנית/טרייד).  
- **טכני:** `notesForm.js` — להסיר "אופציונלי" מהתווית; לחייב `parent_id` כשנבחר `parent_type`; טעינת אופציות (entity option loader) ובחירה ספציפית.
