## רשימת בעיות ותיקונים

### ✅ בעיה #1: כותרות בטבלה מציגות שם משתנה במקום טקסט עברית

**תיאור הבעיה:**
כותרות הטבלה הציגו קוד גולמי כמו `data-button-type="SORT"` במקום טקסט עברית ברור.

**התיקון שבוצע:**
- החלפת כל כותרות הטבלה מכפתורים עם `data-button-type` לכפתורי HTML סטנדרטיים
- הוספת טקסט עברית ברור: חשבון, סוג, סכום, תאריך, תיאור, מקור
- הוספת איקוני מיון (`↕`) לכל כותרת
- קישור לפעולת מיון פונקציונלית עם `window.sortTableData`

**קוד לפני התיקון:**
```html
<th class="col-account"><button data-button-type="SORT" data-classes="btn btn-link" data-text="מיון"></button>", "sortable-header", "style="border: none; background: none; padding: 0; margin: 0; width: 100%; text-align: center; color: inherit; text-decoration: none;"", "חשבון")} </th>
```

**קוד אחרי התיקון:**
```html
<th class="col-account">
    <button class="btn btn-link sortable-header" onclick="if (typeof window.sortTableData === 'function') { window.sortTableData(0, window.cashFlowsData || [], 'cash_flows', window.updateCashFlowsTable); }" style="border: none; background: none; padding: 0; margin: 0; width: 100%; text-align: center; color: inherit; text-decoration: none;">
        חשבון <span class="sort-icon">↕</span>
    </button>
</th>
```

### ⚠️ בעיה #2: חשבון מציג מזהה במקום שם החשבון

**תיאור הבעיה:**
בטבלת תזרימי מזומנים, עמודת החשבון מציגה את המזהה המספרי של החשבון במקום שם החשבון.

**סטטוס**: נדרש טיפול נוסף 🔄

**הערות:**
- הקוד ב-`cash_flows.js` כבר משתמש ב-`getAccountNameById()` כדי לקבל שם חשבון
- הפונקציה מחפשת נתונים ב-`window.HeaderSystem.accountsCache` או `window.trading_accountsData`
- ייתכן שהבעיה היא בנתונים שמגיעים מהשרת או בטעינת נתוני החשבונות

**פעולות נדרשות:**
1. בדיקת מבנה הנתונים שמגיעים מהשרת
2. וידוא שנתוני החשבונות נטענים כראוי
3. בדיקת התאמה בין השדות `trading_account_id` ו-`account_id`

### ✅ בעיה #3: הסרת כפתורי שמור מיותרים

**תיאור הבעיה:**
היו כפתורי "שמור" מיותרים וכפולים בכמה מקומות בעמוד.

**התיקון שבוצע:**

- החלפת כפתורי שמור כפולים במודלים בכפתורים פונקציונליים

**קוד לפני התיקון:**
```html

```

```html

### ✅ בעיה #4: הוספת כפתור הוסף במקום כפתור שמור השני

**תיאור הבעיה:**
חסר כפתור "הוסף" לפתיחת מודל הוספת תזרים מזומנים חדש.

**התיקון שבוצע:**
- הוספת כפתור "הוסף תזרים" בסקשן ניהול תזרימי מזומנים
- כפתור עם איקון Font Awesome (`fas fa-plus`)
- קישור לפונקציה `window.showAddCashFlowModal`

**קוד שהוסף:**

    <i class="fas fa-plus"></i> הוסף תזרים
</button>
```
