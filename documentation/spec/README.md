# אפיון מערכת (Backfilled Spec) – תקציר וניווט

מסמך זה מרכז את מבנה האפיון הבנוי לאחור לפי מצב המערכת בפועל (Backend, Frontend, DB,
Mockups).  
*תקצירים, ראשי פרקים וסיכומים – בעברית; עומק טכני – באנגלית.*

## מטרות

- לאפשר ליזמים/משקיעים לקבל תמונת-על מהירה בעברית.
- לתת לצוותים טכניים מסלול ישיר למסמכי עומק באנגלית לכתיבה/הרחבה/שכתוב.
- לשקף מצב נוכחי (דצמבר 2025), POC ל-web, והבדלי פערים ידועים.

## קהלי יעד

- **מנהלים/יזמים/משקיעים (עברית):** תקציר מנהלים, סיפור הדגמה, מפת דרכים קצרה.
- **צוותים טכניים (אנגלית):** Architecture, Data Model, API, UI Pages & Flows, Runtime/Ops,
  Gaps/Roadmap.

## ניווט מהיר

- תקציר מנהלים (עברית): `executive/EXEC_SUMMARY_HE.md`
- Technical Overview (English): `technical/OVERVIEW.md`
- Architecture: `technical/ARCHITECTURE.md`
- Data Model: `technical/DATA_MODEL.md`
- API Surface: `technical/API_SURFACE.md`
- Frontend Pages & Flows: `technical/FRONTEND_PAGES_AND_FLOWS.md`
- Runtime & Ops: `technical/RUNTIME_AND_OPS.md`
- Gaps & Roadmap: `technical/ROADMAP_AND_GAPS.md`

## היקף וגרסאות

- מקור אמת ל-DB: סביבת פיתוח PostgreSQL (start_server.sh מזהה env לפי תיקייה).
- כיסוי עמודים: כל העמודים לפי `documentation/PAGES_LIST.md`, כולל mockups ודפי daily snapshots.
- מערכות כלליות: שימוש ב-General Systems בלבד (ראה `documentation/frontend/GENERAL_SYSTEMS_LIST.md`).

## איך להשתמש

1. קרא את התקציר בעברית לקבלת ההקשר והמסלול להדגמה.
2. עבור למסמכי האנגלית לפי הצורך (ארכיטקטורה → מודל נתונים → API → UI).
3. בדוק פערים/roadmap לפני החלטות מוצר או מחויבויות ללקוח.
