## 🔍 תרחיש 1: Data Import - טעינת חשבונות מסחר

### מטרה

לבדוק שטעינת חשבונות מסחר בעמוד data_import.html עובדת נכון עם השירות החדש.

### תנאים מוקדמים

1. השרת רץ (`./start_server.sh`)
2. יש חשבונות מסחר במערכת
3. הדפדפן פתוח ב-`http://localhost:8080/data_import.html`

### שלבים

1. פתח את הקונסולה של הדפדפן (F12)
2. בדוק ש-`window.DataImportData` קיים
3. טען את העמוד מחדש (F5)
4. בדוק בקונסולה שהחשבונות נטענו:

   ```javascript
   // בדוק שהשירות קיים
   console.log(window.DataImportData);
   
   // בדוק שהמטמון עובד
   window.UnifiedCacheManager.get('data-import-accounts').then(console.log);
   ```

5. בחר חשבון מהרשימה
6. בדוק שהיסטוריית הייבוא נטענת

### תוצאה צפויה

- ✅ החשבונות נטענים בהצלחה
- ✅ אין שגיאות בקונסולה
- ✅ הנתונים נשמרים במטמון
- ✅ היסטוריית הייבוא נטענת בהצלחה

### בדיקות נוספות

- [ ] בדוק שהמטמון עובד (טעינה שנייה מהירה יותר)
- [ ] בדוק force reload (Ctrl+F5)
- [ ] בדוק טיפול בשגיאות (כבה את השרת ונסה לטעון)

## 🔍 תרחיש 2: Executions - פעולות CRUD

### מטרה

לבדוק שכל פעולות ה-CRUD בעמוד executions.html עובדות נכון עם השירות החדש.

### תנאים מוקדמים

1. השרת רץ
2. יש ביצועים במערכת
3. הדפדפן פתוח ב-`http://localhost:8080/executions.html`

### שלבים

#### 2.1 יצירת ביצוע חדש

1. לחץ על כפתור "הוסף ביצוע" או "Add Execution"
2. מלא את הטופס:
   - Symbol: `AAPL`
   - Quantity: `100`
   - Price: `150.00`
   - Side: `buy`
3. שמור את הביצוע
4. בדוק בקונסולה:

   ```javascript
   // בדוק שהשירות קיים
   console.log(window.ExecutionsData);
   
   // בדוק שהמטמון התעדכן
   window.UnifiedCacheManager.get('executions-data').then(console.log);
   ```

#### 2.2 עדכון ביצוע קיים

1. בחר ביצוע מהרשימה
2. לחץ על "ערוך" או "Edit"
3. שנה את הכמות ל-`150`
4. שמור את השינויים
5. בדוק שהביצוע עודכן ברשימה

#### 2.3 מחיקת ביצוע

1. בחר ביצוע מהרשימה
2. לחץ על "מחק" או "Delete"
3. אשר את המחיקה
4. בדוק שהביצוע נמחק מהרשימה

#### 2.4 טעינת פרטי ביצוע

1. בחר ביצוע מהרשימה
2. לחץ על "פרטים" או "Details"
3. בדוק שהפרטים נטענים נכון

### תוצאה צפויה

- ✅ כל פעולות ה-CRUD עובדות
- ✅ המטמון מתעדכן אחרי כל פעולה
- ✅ אין שגיאות בקונסולה
- ✅ ההודעות מוצגות נכון (הצלחה/שגיאה)

### בדיקות נוספות

- [ ] בדוק שהמטמון מתבטל אחרי כל פעולת CRUD
- [ ] בדוק טיפול בשגיאות (נסה ליצור ביצוע עם נתונים לא תקינים)
- [ ] בדוק שהטבלה מתרעננת אחרי כל פעולה

לבדוק שסנכרון המטמון עובד נכון בין Frontend ל-Backend.

### תנאים מוקדמים

1. השרת רץ
2. יש נתונים במערכת
3. הדפדפן פתוח

### שלבים

1. פתח את הקונסולה של הדפדפן
2. בדוק שהמטמון מתעדכן:

   ```javascript
   // בדוק שהמטמון קיים
   console.log(window.UnifiedCacheManager);
   console.log(window.CacheSyncManager);
   
   // בדוק את המטמון הנוכחי
   window.UnifiedCacheManager.get('executions-data').then(console.log);
   ```

3. בצע פעולת CRUD (יצירה/עדכון/מחיקה)
4. בדוק שהמטמון התעדכן:

   ```javascript
   // בדוק שהמטמון התעדכן
   window.UnifiedCacheManager.get('executions-data').then(console.log);
   ```

5. בדוק בקונסולה של הרשת (Network tab) שיש קריאה ל-`/api/cache-sync/invalidate`

## 🔍 תרחיש 4: Error Handling - טיפול בשגיאות

### מטרה

לבדוק שטיפול בשגיאות עובד נכון בכל השירותים.

### תנאים מוקדמים

1. השרת רץ
2. הדפדפן פתוח

### שלבים

#### 4.1 שגיאת רשת

1. פתח את הקונסולה
2. כבה את השרת (`Ctrl+C` בטרמינל)
3. נסה לטעון נתונים:

   ```javascript
   window.DataImportData.loadTradingAccountsForImport().catch(console.error);
   ```

4. בדוק שהודעת שגיאה מוצגת

#### 4.2 שגיאת API (404/500)

1. הפעל את השרת
2. פתח את הקונסולה
3. נסה לטעון נתונים מ-endpoint לא קיים:

   ```javascript
   // זה אמור להיכשל
   fetch('/api/nonexistent').catch(console.error);
   ```

4. בדוק שהודעת שגיאה מוצגת

#### 4.3 JSON לא תקין

1. פתח את הקונסולה
2. נסה לטעון נתונים עם JSON לא תקין (זה אמור להיכשל אוטומטית)
3. בדוק שהודעת שגיאה מוצגת

### תוצאה צפויה

- ✅ כל השגיאות מטופלות נכון
- ✅ הודעות שגיאה מוצגות למשתמש
- ✅ השגיאות נרשמות ב-Logger

### בדיקות נוספות

- [ ] בדוק שהמטמון לא נפגע בשגיאות
- [ ] בדוק fallback behavior
- [ ] בדוק retry logic (אם קיים)
