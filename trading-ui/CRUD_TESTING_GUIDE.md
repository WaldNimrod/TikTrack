# 🧪 מדריך שימוש בעמוד בדיקות CRUD

## 📋 סקירה כללית

עמוד בדיקות CRUD הוא כלי מקצועי לבדיקת כל הפונקציונליות במערכת TikTrack. העמוד מאפשר לבדוק באופן שיטתי את כל הפעולות CRUD (Create, Read, Update, Delete) בכל הישויות במערכת.

## 🎯 מטרת העמוד

העמוד נועד ל:
- **בדיקה שיטתית** של כל הפונקציונליות במערכת
- **תיעוד בעיות** ובעיות שנמצאו במהלך הבדיקה
- **מעקב אחר התקדמות** הבדיקות
- **אימות חיבור לבסיס הנתונים** מכל הכפתורים והפעולות

## 🚀 איך להתחיל

## 🔧 איך לבצע בדיקה

### שלב 1: בחירת ישות
1. לחץ על הטאב של הישות שברצונך לבדוק
2. העמוד יפתח את הטאב עם כל הבדיקות הרלוונטיות

### שלב 2: פתיחת העמוד הנבדק
1. לחץ על כפתור **"פתח עמוד"** (כחול)
2. העמוד יפתח בטאב חדש
3. **חשוב**: השאר את שני הטאבים פתוחים

### שלב 3: בדיקות בסיסיות
בכל ישות, בדוק תחילה:
- ✅ **עמוד נטען ללא שגיאות**
- ✅ **עיצוב CSS נראה תקין**
- ✅ **אין שגיאות JavaScript בקונסול**
- ✅ **RTL ועברית נכונים**

### שלב 4: בדיקת כפתורים
לכל כפתור ברשימה:
1. לחץ על הכפתור בעמוד הנבדק
2. בדוק שהפעולה מתבצעת נכון
3. סמן ✅ אם הפעולה עובדת
4. סמן ❌ אם יש בעיה

### שלב 5: בדיקות CRUD מפורטות

#### Create (יצירה)
1. לחץ על כפתור "הוסף [ישות] חדש"
2. בדוק שהמודל נפתח
3. מלא את הטופס
4. לחץ שמור
5. בדוק שהנתונים נשמרו בבסיס הנתונים
6. בדוק שהטבלה מתעדכנת

#### Read (קריאה)
1. בדוק שהטבלה נטענת
2. בדוק שהנתונים מוצגים נכון
3. בדוק שהעמודות נכונות
4. בדוק שהנתונים מעודכנים

#### Update (עדכון)
1. לחץ על כפתור "ערוך" בשורה כלשהי
2. בדוק שהמודל נפתח עם הנתונים הקיימים
3. שנה נתונים
4. לחץ שמור
5. בדוק שהשינויים נשמרו בבסיס הנתונים
6. בדוק שהטבלה מתעדכנת

#### Delete (מחיקה)
1. לחץ על כפתור "מחק" בשורה כלשהי
2. בדוק שמופיע אישור מחיקה
3. לחץ "אישור"
4. בדוק שהרשומה נמחקה מבסיס הנתונים
5. בדוק שהטבלה מתעדכנת

### שלב 6: תיעוד בעיות
אם מצאת בעיה:
1. כתב אותה בשדה "הערות ובעיות"
2. תאר בדיוק מה קרה
3. ציין באיזה כפתור/פעולה
4. הוסף צילום מסך אם נדרש

### שלב 7: סיום בדיקה
לאחר שסיימת לבדוק ישות:
- לחץ **"סיימתי בדיקה"** (ירוק) אם הכל עובד
- לחץ **"יש בעיות"** (כתום) אם יש בעיות
